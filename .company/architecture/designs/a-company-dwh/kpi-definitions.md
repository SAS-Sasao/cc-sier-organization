# KPI・メトリクス定義書

**プロジェクト**: A社基幹システム DWH構築
**対象システム**: メダリオンアーキテクチャ Gold層
**作成日**: 2026-03-18
**バージョン**: 1.0

---

## 1. KPI体系の概要

本定義書は、A社の経営・営業分析における KPI・メトリクスを体系的に定義する。
KPIは以下の3階層に分類する。

| ティア | 区分 | 用途 | 対象者 |
|--------|------|------|--------|
| Tier 1 | 経営判断に必須 | 月次・四半期の意思決定に直結するKPI | 経営層 |
| Tier 2 | 運用改善用 | 営業活動の改善・最適化に活用するKPI | 営業マネージャー |
| Tier 3 | 深掘り分析用 | 課題の深堀・仮説検証に使用するKPI | 営業担当者・アナリスト |

## 2. KPI一覧

### 2.1 Tier 1 - 経営判断に必須なKPI

#### KPI-T1-001: 月次受注金額

| 項目 | 内容 |
|------|------|
| **KPI名** | 月次受注金額 |
| **定義** | 当月中に「受注」ステータスに移行した見積の合計金額 |
| **計算式** | `SUM(見積金額) WHERE ステータス = '受注' AND 受注確定日 IN 対象月` |
| **粒度** | 月次 / 部署別 / 担当者別 |
| **データソース** | 見積管理リスト（fact_estimates） |
| **優先度** | Tier 1 |

#### KPI-T1-002: 受注率（勝率）

| 項目 | 内容 |
|------|------|
| **KPI名** | 受注率（Win Rate） |
| **定義** | 対象期間中にクローズされた見積のうち、受注に至った割合 |
| **計算式** | `受注件数 / (受注件数 + 失注件数) × 100` |
| **粒度** | 月次 / 四半期 / 年次 / 担当者別 / 業種別 |
| **データソース** | 見積管理リスト（fact_estimates） |
| **優先度** | Tier 1 |

#### KPI-T1-003: 四半期売上達成率

| 項目 | 内容 |
|------|------|
| **KPI名** | 四半期売上達成率 |
| **定義** | 四半期の受注金額合計を、設定した目標値で除した達成率 |
| **計算式** | `四半期受注金額合計 / 四半期売上目標 × 100` |
| **粒度** | 四半期 / 部署別 |
| **データソース** | 見積管理リスト（fact_estimates）/ 目標マスタ（別途定義） |
| **優先度** | Tier 1 |

#### KPI-T1-004: パイプライン総額

| 項目 | 内容 |
|------|------|
| **KPI名** | パイプライン総額 |
| **定義** | 現在進行中の見積・案件の合計金額（失注・受注済みを除く） |
| **計算式** | `SUM(見積金額) WHERE ステータス NOT IN ('受注', '失注', 'キャンセル')` |
| **粒度** | 時点（日次スナップショット） / 担当者別 |
| **データソース** | 見積管理リスト（fact_estimates）/ 案件管理リスト（fact_deals） |
| **優先度** | Tier 1 |

#### KPI-T1-005: 顧客別売上TOP10

| 項目 | 内容 |
|------|------|
| **KPI名** | 顧客別売上ランキング |
| **定義** | 対象期間中の受注金額を顧客別に集計したランキング |
| **計算式** | `SUM(受注金額) GROUP BY 顧客コード ORDER BY SUM(受注金額) DESC LIMIT 10` |
| **粒度** | 年次 / 四半期 / 顧客別 |
| **データソース** | 見積管理リスト（fact_estimates）/ 顧客マスタ（dim_customer） |
| **優先度** | Tier 1 |

### 2.2 Tier 2 - 運用改善用KPI

#### KPI-T2-001: 見積件数（新規・更新）

| 項目 | 内容 |
|------|------|
| **KPI名** | 月次見積件数 |
| **定義** | 当月中に作成された見積の件数（新規作成日ベース） |
| **計算式** | `COUNT(見積番号) WHERE 作成日 IN 対象月` |
| **粒度** | 月次 / 担当者別 / 顧客別 |
| **データソース** | 見積管理リスト（fact_estimates） |
| **優先度** | Tier 2 |

#### KPI-T2-002: 平均見積金額

| 項目 | 内容 |
|------|------|
| **KPI名** | 平均見積金額（ASP: Average Selling Price） |
| **定義** | 対象期間内に作成された見積の平均金額 |
| **計算式** | `SUM(見積金額) / COUNT(見積番号)` |
| **粒度** | 月次 / 四半期 / 担当者別 / 業種別 |
| **データソース** | 見積管理リスト（fact_estimates） |
| **優先度** | Tier 2 |

#### KPI-T2-003: 見積リードタイム

| 項目 | 内容 |
|------|------|
| **KPI名** | 見積作成リードタイム |
| **定義** | 案件発生（案件登録日）から見積提出（見積作成日）までの経過日数 |
| **計算式** | `AVG(見積作成日 - 案件開始日)` ※案件番号で結合 |
| **粒度** | 月次 / 担当者別 |
| **データソース** | 見積管理リスト（fact_estimates）/ 案件管理リスト（fact_deals） |
| **優先度** | Tier 2 |

#### KPI-T2-004: 受注までのサイクルタイム

| 項目 | 内容 |
|------|------|
| **KPI名** | 見積→受注サイクルタイム |
| **定義** | 見積作成日から受注確定日までの経過日数（受注案件のみ） |
| **計算式** | `AVG(受注確定日 - 見積作成日) WHERE ステータス = '受注'` |
| **粒度** | 月次 / 四半期 / 担当者別 / 業種別 |
| **データソース** | 見積管理リスト（fact_estimates） |
| **優先度** | Tier 2 |

#### KPI-T2-005: 業種別受注率

| 項目 | 内容 |
|------|------|
| **KPI名** | 業種別受注率 |
| **定義** | 顧客の業種ごとに集計した受注率 |
| **計算式** | `受注件数 / (受注件数 + 失注件数) × 100 GROUP BY 業種` |
| **粒度** | 四半期 / 年次 / 業種別 |
| **データソース** | 見積管理リスト（fact_estimates）/ 顧客マスタ（dim_customer） |
| **優先度** | Tier 2 |

#### KPI-T2-006: 担当者別パイプライン件数・金額

| 項目 | 内容 |
|------|------|
| **KPI名** | 担当者別パイプライン |
| **定義** | 担当者ごとの進行中見積の件数と合計金額 |
| **計算式** | `COUNT(見積番号), SUM(見積金額) WHERE ステータス = '進行中' GROUP BY 担当者ID` |
| **粒度** | 時点（日次） / 担当者別 |
| **データソース** | 見積管理リスト（fact_estimates） |
| **優先度** | Tier 2 |

#### KPI-T2-007: フェーズ別案件件数

| 項目 | 内容 |
|------|------|
| **KPI名** | 案件フェーズ別件数 |
| **定義** | 各フェーズに在籍する案件の件数と合計受注予定金額 |
| **計算式** | `COUNT(案件番号), SUM(受注金額) GROUP BY フェーズ` |
| **粒度** | 時点（日次） / フェーズ別 |
| **データソース** | 案件管理リスト（fact_deals） |
| **優先度** | Tier 2 |

### 2.3 Tier 3 - 深掘り分析用KPI

#### KPI-T3-001: フェーズ別平均滞留日数

| 項目 | 内容 |
|------|------|
| **KPI名** | フェーズ滞留日数 |
| **定義** | 各フェーズで案件が滞在した平均日数（フェーズ遷移ログより算出） |
| **計算式** | `AVG(フェーズ終了日 - フェーズ開始日) GROUP BY フェーズ` |
| **粒度** | 月次 / フェーズ別 / 担当者別 |
| **データソース** | 案件管理リスト（fact_deals）/ フェーズ履歴（Silver層） |
| **優先度** | Tier 3 |

#### KPI-T3-002: コンバージョン率（ファネル各段階）

| 項目 | 内容 |
|------|------|
| **KPI名** | ファネルコンバージョン率 |
| **定義** | 各ステータス間の遷移率（見積作成→提出→交渉→受注） |
| **計算式** | `次ステータス移行件数 / 前ステータス件数 × 100` |
| **粒度** | 四半期 / 業種別 / 担当者別 |
| **データソース** | 見積管理リスト（mart_conversion_funnel） |
| **優先度** | Tier 3 |

#### KPI-T3-003: 顧客単価変動率

| 項目 | 内容 |
|------|------|
| **KPI名** | 顧客別単価変動率 |
| **定義** | 前期比での顧客ごとの平均受注単価の変動率 |
| **計算式** | `(当期平均単価 - 前期平均単価) / 前期平均単価 × 100` |
| **粒度** | 四半期 / 年次 / 顧客別 |
| **データソース** | 見積管理リスト（fact_estimates）/ 顧客マスタ（dim_customer） |
| **優先度** | Tier 3 |

#### KPI-T3-004: 失注理由別分析

| 項目 | 内容 |
|------|------|
| **KPI名** | 失注理由別件数・金額 |
| **定義** | 失注した見積を失注理由ごとに集計（失注理由フィールドが必要） |
| **計算式** | `COUNT(見積番号), SUM(見積金額) WHERE ステータス = '失注' GROUP BY 失注理由` |
| **粒度** | 四半期 / 担当者別 |
| **データソース** | 見積管理リスト（fact_estimates） ※失注理由フィールドの追加を推奨 |
| **優先度** | Tier 3 |

#### KPI-T3-005: リピート顧客率

| 項目 | 内容 |
|------|------|
| **KPI名** | リピート受注率 |
| **定義** | 対象期間中に2件以上受注した顧客の比率 |
| **計算式** | `COUNT(DISTINCT 顧客コード WHERE 受注件数 >= 2) / COUNT(DISTINCT 顧客コード WHERE 受注件数 >= 1) × 100` |
| **粒度** | 年次 |
| **データソース** | 見積管理リスト（fact_estimates）/ 顧客マスタ（dim_customer） |
| **優先度** | Tier 3 |

## 4. 目標値設定ガイドライン

### 4.1 設定の基本方針

目標値はシステム側では保持せず、Power BI レポートレベルのパラメータとして管理することを推奨する。
これにより、経営方針の変更に対して柔軟に対応できる。

ただし、目標マスタテーブル（`dim_targets`）を Gold層に配置することで、
前年同期比・予実対比などのダッシュボード表示にも対応できる構成とする。

### 4.2 各KPIの目標値設定の考え方

| KPI | 目標値の考え方 | 推奨設定方法 |
|-----|--------------|------------|
| 月次受注金額 | 前年同月実績の 105〜110% を初期目標として設定 | dim_targets に月別目標金額を格納 |
| 受注率 | 業界平均（製造業向けBtoB: 30〜40%）を参考に社内実績から設定 | Power BI パラメータ（固定閾値） |
| 四半期売上達成率 | 100% を目標（80% 以下は警告、90% 以下は注意） | Power BI 条件付き書式で視覚化 |
| パイプライン総額 | 四半期売上目標の 3〜4倍 を健全なパイプラインの目安とする | 計算列として Power BI 側で保持 |
| 見積→受注サイクルタイム | 現状の中央値を基準に 10〜20% 短縮を目標とする | 初期値は実績データから自動算出 |
| フェーズ滞留日数 | フェーズごとに SLA（上限日数）を設定し、超過を警告表示 | dim_targets の phase_sla_days を参照 |

### 4.3 アラート閾値の設定例

```
受注率:
  - 緑（良好）: 35% 以上
  - 黄（注意）: 25% 以上 35% 未満
  - 赤（警告）: 25% 未満

四半期達成率:
  - 緑（良好）: 90% 以上
  - 黄（注意）: 75% 以上 90% 未満
  - 赤（警告）: 75% 未満

パイプライン充足率 (パイプライン総額 / 四半期目標):
  - 緑（良好）: 300% 以上
  - 黄（注意）: 200% 以上 300% 未満
  - 赤（警告）: 200% 未満
```

## 5. データ品質ルール

各 KPI の計算において、以下のデータ品質チェックを Silver→Gold 変換時に適用する。

| チェック項目 | ルール |
|------------|--------|
| 見積金額 | NULL および 0円 以下の値は除外し、データ品質ログに記録 |
| ステータス値 | 許容値リスト（受注/失注/進行中/保留/キャンセル）以外は「不明」に変換 |
| 日付 | 作成日 > 更新日 の矛盾レコードはフラグを立て、更新日を作成日で上書き |
| 担当者ID | 担当者マスタに存在しない社員番号は「未割当」として処理 |
| 顧客コード | 顧客マスタに存在しない顧客コードは「不明顧客」として処理し、経営層向けには除外 |
