# 日次ダイジェスト 品質チェックリスト

秘書がダイジェスト生成後、コミット前に必ず実行すること。
`severity_on_fail: error` のため、未達項目があればコミットをブロックする。

## 必須項目

### ソースリンク
- [ ] 全記事タイトルがマークダウンリンク `[タイトル](URL)` 形式になっている
- [ ] リンクなしの記事（`**タイトル**` のみ）が存在しない
- [ ] URLが `https://` で始まっている（相対パス不可）

### 構成
- [ ] ヘッダーに組織名・生成方式・巡回ソース数が記載されている
- [ ] 「本日のハイライト」セクションが存在する
- [ ] 技術セクション（A章）と小売セクション（B章）の両方が存在する
- [ ] 巡回メタデータ（D章）に全ソースのステータスが記載されている

### 巡回網羅性
- [ ] 優先度「高」ソースが全て巡回されている（スキップ時は理由を明記）
- [ ] 各ソースから最低1件の記事が収集されている

## 推奨項目

- [ ] クロスドメイン分析（C章）にSIer示唆が含まれている
- [ ] 重複記事が統合されている（同一記事が複数ソースに掲載された場合）
- [ ] 記事数の合計が巡回メタデータの総記事数と一致している
