# 日次ダイジェスト 品質チェックリスト

秘書がダイジェスト生成後、コミット前に必ず実行すること。
`severity_on_fail: error` のため、未達項目があればコミットをブロックする。

## 必須項目

### ソースリンク
- [ ] 全記事タイトルがマークダウンリンク `[タイトル](URL)` 形式になっている
- [ ] リンクなしの記事（`**タイトル**` のみ）が存在しない
- [ ] URLが `https://` で始まっている（相対パス不可）

### 構成
- [ ] ヘッダーに組織名・生成方式・巡回ソース数が記載されている
- [ ] 「本日のハイライト」セクションが存在する
- [ ] 技術セクション（A章）と小売セクション（B章）の両方が存在する
- [ ] 巡回メタデータ（D章）に全ソースのステータスが記載されている

### 巡回網羅性
- [ ] 優先度「高」ソースが全て巡回されている（スキップ時は理由を明記）
- [ ] 各ソースから最低1件の記事が収集されている

## 推奨項目

- [ ] クロスドメイン分析（C章）にSIer示唆が含まれている
- [ ] 重複記事が統合されている（同一記事が複数ソースに掲載された場合）
- [ ] 記事数の合計が巡回メタデータの総記事数と一致している

---

## フォーマットテンプレート

**重要**: 以下のフォーマットに必ず従うこと。ソース別ではなくテーマ別に分類し、記事はテーブル形式で記載する。

### ヘッダー

```markdown
# 日次ダイジェスト YYYY-MM-DD

> **組織**: ドメイン知識や技術スタック収集PJT (`domain-tech-collection`)
> **生成方式**: {方式}（wf-daily-digest）
> **巡回ソース数**: {N}件（{取得結果}）
> **オペレーター**: {operator}
```

### 章構成（順序固定）

| 章 | セクション名 | 内容 |
|----|------------|------|
| ハイライト | 本日のハイライト | 1〜5件の注目トピック。番号付きリスト |
| A章 | 技術スタック | テーマ別サブセクション（A1〜A6） |
| B章 | 小売ドメイン | テーマ別サブセクション（B1〜B5） |
| C章 | クロスドメイン分析 | 技術×小売のSIer示唆。パラグラフ形式で3〜5トピック |
| D章 | 巡回メタデータ | ソース一覧テーブル。ステータス・記事数・備考 |

### A章（技術）のサブセクション

```
### A1. AI駆動開発・エージェント
### A2. AI・ML・LLM
### A3. クラウド・インフラ
### A4. データ基盤・DWH
### A5. 開発プラクティス・設計
### A6. セキュリティ
```

該当記事がないサブセクションは省略可。

### B章（小売）のサブセクション

```
### B1. 業態変革・新店
### B2. 経営・人事戦略
### B3. PB・商品戦略
### B4. EC・デジタル
### B5. 決算・統計
### B6. セキュリティ
```

該当記事がないサブセクションは省略可。

### 記事テーブル形式（必須）

```markdown
| # | 記事 | ソース | 要約 |
|---|------|--------|------|
| 1 | [記事タイトル](https://example.com/path) | ソース名 | 1行の要約 |
```

- 記事タイトルは `[タイトル](URL)` のマークダウンリンク形式（必須）
- ソース名は「流通ニュース」「DevelopersIO」等の短縮名
- 要約は1行（句読点で終わる）
- **リスト形式（`- [タイトル](URL) — 要約`）は使用しない**

### C章（クロスドメイン分析）の形式

```markdown
### 1. {テーマ名}
{パラグラフで分析内容を記述。技術側の動向と小売側の動向を結びつけ、SIer示唆を述べる。}
```

- **テーブル形式ではなく、パラグラフ形式で記述する**
- 3〜5トピック

### D章（巡回メタデータ）の形式

```markdown
| # | ソース | ステータス | 取得記事数 | 備考 |
|---|--------|-----------|-----------|------|
| 1 | 流通ニュース | ✅ 成功 | 10件 | 備考テキスト |
```

末尾に `**総記事数**: 技術{N}件 + 小売{N}件 = {合計}件` を記載。
