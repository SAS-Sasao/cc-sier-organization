- **ファイル名**: talent-data-arch
- **図の種類**: C4モデル
- **案件名**: 人材データ統合
- **作成日**: 2026-03-29
- **作成者**: SAS-Sasao
- **ツール**: open_drawio_xml

## 概要

組織内メンバーや契約パートナーの情報を管理アプリ/SharePointで一元管理し、
MDMで名寄せ・クレンジングを行った上で、データレイク/DWHに蓄積。
BIツールで可視化し、タレントマネジメントアプリで人材・案件マッチングを分析・提示する。

## 構成（5層）

1. **ソースシステム層**: メンバー/パートナー管理アプリ（内製）、SharePoint（外部）
2. **データ統合層**: MDM（名寄せ・クレンジング・ゴールデンレコード生成）
3. **データ蓄積層**: データレイク（生データ）、DWH（構造化データ）
4. **分析・可視化層**: BIツール（稼働率・スキル分布の可視化）
5. **活用層**: タレントマネジメントアプリ（人材・案件マッチング）

## アクター

- **HR担当者**: メンバー/パートナー情報の登録・管理
- **PM / リソース管理者**: タレントマネジメントアプリで人材検索・案件分析

## データフロー

```
HR → メンバー管理アプリ / SharePoint
  → MDM（名寄せ・クレンジング）
    → データレイク（生データ蓄積）
    → DWH（マスタデータ同期）
      → BIツール（分析インサイト）
        → タレントマネジメントアプリ ← PM
      → タレントマネジメントアプリ（マスタデータAPI）
データレイク → DWH（ETL/ELT）
```
