## 2. 事業内容・担当システム

### LDIの事業ドメイン

LDIの事業内容は「ローソン事業のデジタル化推進およびローソン次世代システムの設計・開発・導入・運用」と定義されている。

### LDIが内製化している主要領域

| 領域 | 内容 | 技術 |
|------|------|------|
| **ローソンアプリ** | 2,000万DL超のコンシューマー向けアプリ（iOS/Android） | Swift, Kotlin, Clean Architecture + MVVM |
| **Webフロントエンド** | アプリ内WebView等のWeb領域 | Next.js, Amplify, AppSync, DynamoDB |
| **サーバーサイド** | アプリ・Webのバックエンドサービス | Go (Gin), Clean Architecture |
| **データサイエンス** | 販売予測、顧客分析、地域限定商品の空間解析 | Python, R, SAS |
| **IoTアプリケーション** | 店舗IoTデバイス連携のPoC | Raspberry Pi, Python |
| **Flutter開発** | ネイティブアプリの一部 | Flutter |

### LDIの組織体制（推定）

テックブログ・採用情報から読み取れる部署構成:
- **コンシューマーサービス部**: ローソンアプリ（iOS/Android）の開発。スクラム開発を実践
- **インフラ部門**: クラウド基盤の運用・維持管理、新規システム開発環境の設計・構築
- **データサイエンス部門**: 販売データ分析、AI/ML関連
- **品質管理部門**: テスト戦略、品質保証
- **セキュリティ部門**: 情報セキュリティ対策

### 現在の採用ポジション（10職種）

1. プロジェクトマネジャー
2. システムエンジニア（Web/基幹システム担当）

## 4. シグマクシスとの役割分担

### 設立時（2016年）の想定

| 役割 | ローソン側 | シグマクシス側 |
|------|-----------|--------------|
| 出資比率 | 66% | 34% |
| 提供リソース | 業務知識、店舗ネットワーク、既存システム知見 | PMノウハウ、デジタルテクノロジー知見、コンサル力 |
| 狙い | 内製化推進による開発スピード向上 | コンサル会社としての実績・知見の提供 |

### 完全子会社化後（2020年〜）

2020年4月にシグマクシスが全株式をローソンに譲渡し、LDIはローソンの完全子会社となった。これはLDIの内製化路線が軌道に乗り、シグマクシスの「立ち上げ支援」の役割が完了したと解釈できる。

- シグマクシスのPMノウハウはLDI内部に移転・定着済み
- LDI独自の開発文化・プロセスが確立されたことで独立運営が可能に
- 出資関係は解消されたが、コンサル案件としての協業は継続している可能性あり

### ローソンのIT開発パートナーマップ

| パートナー | 担当領域 | 関係性 |
|-----------|---------|--------|
| **LDI（内製）** | ローソンアプリ、Web系、データ分析、IoT PoC | ローソン完全子会社（内製チーム） |
| **NECプラットフォームズ** | POS端末ハードウェア・ファームウェア（共同開発） | ハードウェアパートナー |
| **NEC** | セルフレジ、顔認証、映像解析AI、深夜省人化実験 | AI・ハードウェアパートナー |
| **フューチャーアーキテクト** | 統合マスタシステム、Loppi、セミオート発注→AI.CO | 基幹系SIパートナー（2007年〜） |
| **日立ソリューションズ** | AWSクラウド基盤設計・構築・運用、AppDynamics | クラウドインフラパートナー |
| **NSSOL（日鉄ソリューションズ）** | マネージドクラウド「absonne」（既存システム移行） | マルチクラウドパートナー |
| **アイレット（iret/cloudpack）** | 公式Webサイト群のAWS移行、セキュリティ | WebインフラSIパートナー |

| **KDDI** | 通信インフラ、WAKONX（AI基盤）、Real×Tech | 経営パートナー（議決権50%） |

### 内製 vs 外注の境界線（推定）

```
[LDI内製領域]
  ├── コンシューマー接点（アプリ、Web）
  ├── データ分析・サイエンス
  ├── IoT PoC
  └── 先進技術検証（生成AI等）

[外注（SIer委託）領域]
  ├── 基幹系システム（フューチャーアーキテクト）
  │   ├── 統合マスタシステム
  │   ├── Loppiシステム
  │   └── AI.CO（発注AI）
  ├── POS/レジ端末（NECプラットフォームズ）
  ├── クラウドインフラ（日立ソリューションズ、NSSOL）
  ├── AI・顔認証（NEC）
  └── 店舗DX実証（Microsoft、KDDI）
```

### LDI vs NECの棲み分け（推定）

| 軸 | LDI | NEC |
|----|-----|-----|
| **レイヤー** | アプリケーション層（モバイルアプリ、Web） | ハードウェア層 + AI層 |
| **対象** | コンシューマー接点、データ分析 | POS端末、セルフレジ、顔認証、映像解析 |
| **開発スタイル** | アジャイル、内製、スクラム | 受託開発、ハードウェア設計 |
| **技術ドメイン** | Swift/Kotlin/Go/Next.js | 組込み、画像認識AI、生体認証 |
| **クラウド** | AWS上のアプリ開発 | オンプレ+クラウドのハイブリッド |

**重要な示唆**: ローソンにおけるNECの役割は、セブン-イレブンにおけるNECの役割（フルクラウド店舗システムの構築、PMO参画、発注システム開発）と比べて、**ハードウェア・AI特化**であり、基幹系SIの中心はフューチャーアーキテクトが担っている。

## PMとしての示唆

### 1. NECの立ち位置を正確に把握する

ローソンにおけるNECの役割は、セブン-イレブン案件（フルクラウド店舗システムの中核SI）とは大きく異なる。ローソンではNECは**POS端末・セルフレジ等のハードウェア + AI技術**のパートナーであり、基幹系SIの中心はフューチャーアーキテクトが長年担っている。ストアコンピューターのAWS移行案件がNECに発注された場合、それはNECにとってローソン案件の中でSI領域への拡大を意味する可能性がある。

### 2. LDIとの協業ラインを見極める

LDIは約123名の内製チームで、主にコンシューマー向け（アプリ・Web）を担当。クラウドインフラやAI/ML基盤は外部パートナーに委託している。ストコンAWS移行は**LDIの担当外**（インフラ・基幹系領域）である可能性が高く、LDIとの直接的な競合よりも**連携・協調**が求められる。

### 3. フューチャーアーキテクトとの関係に注意

フューチャーアーキテクトは2007年からローソンの基幹系SIを担当する長年のパートナー。統合マスタ、Loppi、AI.CO等の主要システムを握っている。ストコン移行においても、マスタデータ連携やAI.COとの統合が必要になるため、フューチャーアーキテクトとの協調が必須。

### 4. マルチクラウド環境への理解

ローソンはAWS（新規開発）+ NSSOL absonne（既存移行）のマルチクラウド戦略を採用。さらにAzure（IoT）やKDDI WAKONX（AI基盤）も利用。ストコン移行では既存システムとの接続・データ連携において、このマルチクラウド環境の理解が重要。

### 5. セブン-イレブン案件との比較学習

セブン-イレブンではNECが発注システム開発、PMOを担当し、Google Cloud上でフルクラウド・マイクロサービス化を実現。ローソン案件でも同様のアーキテクチャ（クラウドネイティブ、マイクロサービス）が期待される可能性がある。セブン案件の知見をローソン案件に転用できるかが、NEC内での差別化ポイントになり得る。
