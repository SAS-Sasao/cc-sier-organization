### 1. ストコンの全体像（§1）

- [ ] ストコンの定義・役割を自分の言葉で説明できるようにする
  - 店舗運営の中核コンピューター、全国各店舗に1台設置
  - 6つの役割カテゴリ: 発注管理、在庫管理、売上管理、本部通信、サービス管理、従業員管理
- [ ] コンビニ3層構造を理解する
  - 本部システム層（商品マスタ、EOS/EOB、DWH）
  - 店舗システム層（ストコン、POS、マルチメディア端末、決済端末）← ストコンがハブ
  - 物流・サービス層（配送センター、収納代行、チケット発券）
- [ ] ストコンの歴史的経緯を把握する
  - 1978年: EOS導入 → 1985年: ターミナル型端末 → 2000年代: 常時接続 → 2020年代: クラウド移行本格化
- [ ] 主要チェーンのストコン事情を整理する
  - セブン: SIS/ROSS（NEC）、AI発注支援 → AWS本格化
  - ファミマ: Microsoft/Azure提携、Thin Client化推進
  - ローソン: KDDI連携、ローカル5G・エッジ活用のスマートストア構想
  - ミニストップ: イオングループ共通クラウド（AWS）統合

### 2. 主要業務プロセス — 発注（§2-1）

- [ ] 発注3方式を理解する
  - 自動発注（オート）: システムが自動算出、特需に弱い
  - 推奨発注（スマート）: システム提案＋店員判断、最もバランス良い
  - 手動発注: 新商品・イベント商品用
- [ ] 発注の主要パラメータを覚える
  - 発注点（リオーダーポイント）、発注量（EOQ）、リードタイム、棚割数量（フェイス数）、最小発注単位（MOQ）
- [ ] 発注締め時刻と配送区分を把握する
  - 朝便（前日20-22時締め）、昼便（当日6-8時締め）、夕便（当日10-12時締め）

### 3. 主要業務プロセス — 廃棄・在庫・精算（§2-2〜2-4）

- [ ] 廃棄管理フローを理解する
  - 期限確認 → アラート → 廃棄スキャン → 記録 → 本部送信 → 会計計上
  - 重要概念: ロス、廃棄ロス率、チャンスロス、デイリー商品
- [ ] 売上精算（日次精算）フローを理解する
  - POS売上集計 → 日計表作成 → 現金実査 → 突合 → EOB処理
  - 精算書の項目: 総売上、商品別売上、粗利、廃棄ロス、客数・客単価、電子マネー別
- [ ] 在庫管理の概念を整理する
  - 理論在庫 vs 実在庫、在庫差異（ロス）
  - 在庫回転率が極めて高い（デイリー品は1日以内）
  - 温度帯別管理（常温・冷蔵・冷凍・ホット）

### 4. 店舗内システム構成（§3）

- [ ] ストコンと周辺機器の接続構成を理解する
  - POS連携（売上データ集約、商品マスタ配信）
  - マルチメディア端末（Loppi、Famiポート等）
  - 電子マネー・決済端末
  - 温度管理システム（冷蔵ケースセンサー）
- [ ] 本部通信方式を理解する
  - EOS（Electronic Ordering System）: 発注データの送信
  - EOB（Electronic Order Book）: 日次バッチ処理（精算・在庫データの送受信）
  - 通信スケジュール: 上り/下り通信のタイミング
- [ ] ストコンのオフライン耐性を理解する
  - ネットワーク障害時のローカル処理継続
  - データのローカルDB保持＋復旧後の差分同期

### 5. 最新動向（2024〜2026年トレンド）

- [ ] クラウド移行の業界トレンドを把握する
  - 共通トレンド: 「決済・EOB=エッジ、分析・AI=クラウド」のハイブリッド構成
  - MACH アーキテクチャ（Microservices/API-first/Cloud-native/Headless）
  - ストコン機能の単機能API化（発注API、棚割API等）
- [ ] エッジコンピューティングの店舗活用を理解する
  - AWS Outposts / Azure Stack Edge の実証実験
  - カメラ×エッジAIによる欠品検知（NVIDIA Jetson等）
  - ESL（電子棚札）との自動連携
- [ ] コンビニ業界の最新課題を整理する
  - セルフレジ全国展開（酒・タバコ規制で完全無人化は限定的）
  - AI需要予測: 気象・イベント・地域特性の多変量モデル、XAI要件
  - キャッシュレス比率60〜70%、決済規格乱立への対応
- [ ] PCI DSS v4.0 の変更点を理解する（PM視点）
  - 2025年3月末v3.2.1終了、2026年3月末に将来日付要件
  - 主要変更: カスタマイズドアプローチ / WAF必須化 / MFA全面適用
  - コンビニ特有: ATM・マルチメディア端末・収納代行でスコープが広い
