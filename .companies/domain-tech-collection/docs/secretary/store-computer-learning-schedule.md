# ストアコンピューター クラウド移行案件 — 学習スケジュール

**作成日**: 2026-03-21
**作成者**: 秘書室（tech-researcher + retail-domain-researcher の調査結果を統合）
**対象者**: 笹尾豊樹
**案件概要**: コンビニエンスストアのストアコンピューターをオンプレミス→AWSクラウドに移行 + 機能刷新
**参画予定**: 2026年6月

---

## 全体方針

学習領域を **ドメイン知識**（コンビニ業務・ストコン）と **技術スタック**（AWS・移行技術）の2軸で整理し、3月下旬〜5月末の約10週間で段階的に習得する。

```
Week 1-3   ドメイン基礎 + AWS基礎（並行して両方の土台を作る）
Week 4-6   ドメイン深掘り + AWS設計パターン（案件に直結する知識）
Week 7-9   移行技術 + 統合理解（ドメイン×技術を結びつける）
Week 10    総仕上げ + SAA受験
```

---

## Phase 1: 基礎固め（3月下旬〜4月上旬 / Week 1-3）

### ドメイン知識

| 週 | 学習テーマ | 学習内容 | リソース | 目安時間 |
|----|----------|---------|---------|---------|
| W1 | ストコンの全体像 | ストコンの定義・役割・歴史、コンビニ3層構造（本部-店舗-物流）、主要チェーンのシステム事情 | 調査レポート §1 + JFA統計 + 各社IR資料 | 5h |
| W2 | 主要業務プロセス | 発注3方式（自動/推奨/手動）、廃棄管理、日次精算、検品・入荷フロー | 調査レポート §2 + §4 | 6h |
| W3 | 店舗内システム構成 | POS連携、マルチメディア端末、電子マネー・決済、温度管理、本部通信（EOS/EOB） | 調査レポート §3 | 5h |

**到達目標**: ストコンが何をしていて、どんなデータが流れているか説明できる

### 技術スタック

| 週 | 学習テーマ | 学習内容 | リソース | 目安時間 |
|----|----------|---------|---------|---------|
| W1 | AWS基礎 | AWSアカウント作成、IAM、VPC、EC2、S3 の基本操作 | AWS Skill Builder: Cloud Practitioner Essentials（無料） | 8h |
| W2 | データベース基礎 | RDS(Aurora)、DynamoDB の概念と使い分け、ElastiCache | Udemy SAA コース（DB章）+ AWSハンズオン | 6h |
| W3 | コンテナ基礎 | Docker基礎、ECS/Fargate の概念、Lambda 入門 | AWS ハンズオン + Docker公式ドキュメント | 6h |

**到達目標**: AWSの主要サービスを手を動かして触った経験がある

---

## Phase 2: 案件直結知識（4月中旬〜5月上旬 / Week 4-6）

### ドメイン知識

| 週 | 学習テーマ | 学習内容 | リソース | 目安時間 |
|----|----------|---------|---------|---------|
| W4 | データフローの深掘り | 上り/下り通信の詳細、通信スケジュール、バッチ処理タイミング（EOB） | 調査レポート §3-3, §4-3 | 4h |
| W5 | 非機能要件の理解 | 24/365可用性、PCI DSS、食品衛生法、オフライン耐性の要件 | 調査レポート付録 + PCI SSC公式 | 5h |
| W6 | 業界用語の定着 | 用語集の通読、単品管理・棚割・フェイスなどの概念を自分の言葉で説明する練習 | 調査レポート §5 + JFA用語集 | 3h |

**到達目標**: 案件会議で飛び交う業界用語・業務概念が分かる

### 技術スタック

| 週 | 学習テーマ | 学習内容 | リソース | 目安時間 |
|----|----------|---------|---------|---------|
| W4 | 大規模同時接続設計 | API Gateway + SQS によるバックプレッシャー、Auto Scaling、DynamoDB On-Demand | 技術レポート §3-1, §3-2 | 6h |
| W5 | バッチ処理 + リアルタイム処理 | AWS Batch、Step Functions、Kinesis、EventBridge の使い分け | 技術レポート §3-2, §3-3 + AWSハンズオン | 6h |
| W6 | セキュリティ + ネットワーク | Direct Connect、WAF、KMS、GuardDuty、Secrets Manager | 技術レポート §2-4, §2-5 + Well-Architected セキュリティピラー | 6h |

**到達目標**: ストコンのワークロードに合ったAWSサービスの選定理由を説明できる

---

## Phase 3: 移行技術 + 統合（5月中旬〜下旬 / Week 7-9）

### 移行パターン

| 週 | 学習テーマ | 学習内容 | リソース | 目安時間 |
|----|----------|---------|---------|---------|
| W7 | 移行戦略（7R）| Rehost / Replatform / Refactor の使い分け、ストラングラーフィグパターン | 技術レポート §4 | 5h |
| W8 | 移行ツール | DMS、SCT、MGN、Migration Hub、DataSync の概要と使い方 | 技術レポート §4-3 + AWS公式ドキュメント | 6h |
| W9 | 運用・監視・IaC | CloudWatch、X-Ray、CI/CD（CodePipeline）、CDK / Terraform | 技術レポート §5 + AWSハンズオン | 6h |

**到達目標**: オンプレ→AWSの移行ステップを具体的にイメージできる

### ドメイン×技術の統合

| 週 | 学習テーマ | 学習内容 | リソース | 目安時間 |
|----|----------|---------|---------|---------|
| W7 | オフライン耐性設計 | ストコンのネットワーク障害時パターン（ローカルDB + 差分同期）をAWSで設計 | 技術レポート §3-5 | 3h |
| W8 | 店舗-クラウド通信設計 | API Gateway + ALB + ECS/Lambda のアーキテクチャ、WebSocket API | 技術レポート §3-4 | 3h |
| W9 | コンビニ特有の非機能要件 × AWS | 可用性99.99%（Multi-AZ）、PCI DSS（WAF+KMS）、温度データ保管（Timestream/S3） | 両レポートの付録・非機能要件セクション | 3h |

**到達目標**: 「コンビニのこの業務をAWSでこう実現する」を語れる

---

## Phase 4: 総仕上げ（5月末〜6月初 / Week 10）

| テーマ | 学習内容 | 目安時間 |
|-------|---------|---------|
| SAA試験対策 | 模擬試験を解く → 弱点分野を補強 → 受験 | 10h |
| ケーススタディ | ローソン・ファミマ等のAWS活用事例を読み込む | 3h |
| Well-Architected レビュー | 6つのピラーをストコン案件に当てはめて自己チェック | 3h |
| 参画準備 | チーム構成・開発ルール・既存システム概要の事前キャッチアップ（資料があれば） | 4h |

**到達目標**: SAA取得 + 案件初日から議論に参加できる準備が整う

## 学習時間サマリー

| Phase | 期間 | ドメイン | 技術 | 合計 |
|-------|------|---------|------|------|
| Phase 1（基礎） | W1-3（3月下旬〜4月上旬） | 16h | 20h | 36h |
| Phase 2（直結知識） | W4-6（4月中旬〜5月上旬） | 12h | 18h | 30h |
| Phase 3（移行+統合） | W7-9（5月中旬〜下旬） | 9h | 17h | 26h |
| Phase 4（仕上げ） | W10（5月末〜6月初） | 3h | 17h | 20h |
| **合計** | **10週間** | **40h** | **72h** | **112h** |

> 週あたり約11時間（平日1.5h + 週末2.5h程度）のペースで到達可能
