## 1. 調査サマリー

### 1-1. 三社の DX 戦略の基本構造

コンビニ大手3社は、それぞれ親会社・資本構造の影響を強く受けた独自のDX戦略を展開している。2024〜2026年における最大の変化は「IT部門の内製化」と「AI・クラウドの実用段階への移行」である。

| チェーン | DX戦略の軸 | クラウド戦略 | 内製化体制 |
|---------|-----------|------------|---------|
| **セブン-イレブン** | グループ横断デジタル経済圏の構築・Google Cloud基盤での全社DX | Google Cloud（2022年契約、段階移行中） | セブン&アイHD傘下のデジタル統合推進室 |
| **ファミリーマート** | 伊藤忠グループデータ連携・店頭メディア化・Azure AI活用 | Microsoft Azure（2022年戦略提携） | IT本部の内製化強化（外注削減方針） |
| **ローソン** | KDDI連携×Real×Tech構想・LDI内製化・マルチクラウド | AWS（新規開発）+ KDDI WAKONX + NSSOL absonne（既存） | LDI（ローソンデジタルイノベーション、2016年設立・2020年完全子会社化） |

### 1-2. 2024〜2026年の業界全体トレンド

1. **AI発注・需要予測の実用化完了**: 2024年までに大手3社すべてがAI発注を全店展開。フロンティアは精度向上・説明可能AIへ移行
2. **セルフレジの全店化フェーズ**: 2024〜2026年が展開期。人手不足対応と同時に顧客体験向上が目的化
3. **生成AI活用の開始**: 2024〜2025年からSV業務・商品開発・加盟店サポートへ生成AI適用が本格化
4. **アプリ・ポイント経済圏の競争激化**: セブンアプリ（5,000万DL超）を筆頭に、ファミペイ・楽天ポイント対応のローソンが競合
5. **食品ロス対応の制度化**: 値引き自動化・ダイナミックプライシングのシステム対応が規制要件化

## 3. セブン-イレブンのDX戦略

### 3-1. 経営体制とDXの背景

セブン&アイ・ホールディングスは、2024〜2025年にかけてコンビニ事業への集中戦略を明確化した。スーパーマーケット事業（イトーヨーカドー）の縮小・売却方針を打ち出し、国内外のセブン-イレブン事業にデジタル投資を集中させる方針を発表。

2024年には米カナダのAlimentation Couche-TardによるセブンAHDへの買収提案が話題になったが最終的に不成立。これを受けてMBO（経営陣による買収）も検討され、コンビニ事業の競争力強化が至上命題となっている。

### 3-2. Google Cloud戦略提携の全容

2022年に発表したGoogle Cloudとの戦略的提携は、コンビニIT史上最大規模の変革である。

| 領域 | 具体的な取り組み |
|------|--------------|
| **AI需要予測・発注最適化** | Google Cloud AI Platform（Vertex AI）を使った需要予測モデルの再構築。天気・イベント・地域属性・購買トレンドを組み合わせた多変量モデル |
| **DWH・データ基盤** | BigQueryへの全店POSデータ・発注データ集約。従来のオンプレDWHを段階的に移行 |
| **SV業務支援** | 生成AI（Gemini）を活用した店舗訪問前の異常検知・レポート自動生成。SVが管轄する7〜12店の状況をAIが事前整理 |
| **セブンアプリ高度化** | Google Cloud上での会員データ分析・パーソナライズクーポン配信。5,000万DL突破のアプリ基盤を刷新 |
| **グローバル標準化** | 米国7-ElevenのIT基盤との技術共有。グローバルで統一されたデータモデルの整備 |

### 3-3. 店舗DX施策

| 施策 | 状況（2024〜2026年） |
|------|------------------|
| **セルフPOS** | 2025年末までに全国約21,500店への展開目標。フルセルフ型・セミセルフ型の並行展開 |
| **AI発注（ROSS後継）** | ROSS（Retail Operation Support System）をGoogle Cloud基盤に移行中。廃棄削減率15〜20%達成報告（2023年） |
| **7NOW（EC・配送）** | コンビニ在庫のオンデマンド配送サービス。都市部中心に拡大。EC売上の伸長が全体KPIに |
| **セブン銀行ATM活用** | 約27,000台のATMネットワークを活用した金融サービス拡張。スマホATM・外国人対応多言語化 |
| **電子棚札（ESL）** | 一部店舗で実証。価格変更作業の自動化。ストコンからの価格マスタ更新でESLが連動更新 |
| **映像AI（棚前解析）** | 棚カメラ+AIによる欠品検知・顧客行動分析の全店展開検討中 |

### 3-4. データ活用・AI活用

```
[セブン-イレブン データ活用の構造]

全国21,500店舗のPOSデータ（1日数億レコード）
          ↓
   BigQuery（Google Cloud）
   全店売上・発注・廃棄・在庫データを集約
          ↓
   Vertex AI（需要予測・推奨発注）
   気象データ・イベント情報・地域特性を組み合わせ
          ↓
   セブンアプリ（5,000万DL）× 会員購買データ
   個人向けパーソナライズクーポン・商品推薦
          ↓
   SVダッシュボード（生成AIレポート）
   生成AI（Gemini）が店舗訪問前に異常を自動検知・整理
```

### 3-5. 競合との差別化ポイント

- **スケールメリット**: 国内21,500店（2位ファミマの1.3倍）のデータ量が需要予測AIの精度を圧倒的に高める
- **グローバル資産活用**: 19カ国で展開する7-Elevenグループのデータと知見を共有できる唯一の国内チェーン
- **Google Cloudとの排他的連携**: 単なるクラウド利用ではなく、Google側がリソースを投入した共同開発体制
- **セブン銀行の金融データ**: ATM利用履歴・金融行動データとの組み合わせによる顧客理解の深さ

## 4. ファミリーマートのDX戦略

### 4-1. 経営体制と伊藤忠グループ連携

ファミリーマートは伊藤忠商事が主要株主（議決権の過半数保有）。非上場化した2020年以降、短期株価プレッシャーなく中長期DX投資を進める体制を確立している。

伊藤忠グループの事業ポートフォリオ（繊維・化学・食料・生活資材・エネルギー・金融）とのデータ連携が差別化の軸となる。

### 4-2. Microsoft Azure戦略提携

2022年発表のMicrosoftとの包括的戦略提携は、以下の領域で具体化されている。

| 領域 | Azure活用の内容 |
|------|--------------|
| **AI発注・廃棄削減** | Azure OpenAI Serviceを活用した需要予測。廃棄率削減をKPIに設定し、加盟店収益改善に直結 |
| **ファミペイ基盤** | Azure上での会員管理・決済処理・ポイント管理。2,000万超のファミペイユーザー基盤 |
| **店頭サイネージ（メディア事業）** | Azure Media Servicesを活用したデジタルサイネージ配信。全国16,500店舗を「メディア」として広告収益化 |
| **店舗IoT** | Azure IoT Hubによる冷蔵ケース温度管理・設備監視の集中管理 |
| **生成AI（Copilot）** | Microsoft Copilotの業務適用。加盟店サポート（SV業務）・商品開発・マーケティング企画に活用 |

### 4-3. 「ファミリーマートはメディアである」戦略

2022年から本格化した店頭デジタルサイネージ広告事業（「ファミリーマートメディア」）は、小売DXの中で最も独自性の高い施策。

| 指標 | 内容 |
|------|------|
| サイネージ設置店舗数 | 2025年時点で約12,000店（全16,500店の約73%） |
| 1日の広告視聴者数 | 全国推計で約1,000万人超（POPディスプレイとして業界最大級） |
| 広告主 | 食品・飲料・日用品メーカーを中心に数百社 |
| データ連携 | ファミペイ購買データとサイネージ表示履歴の組み合わせで広告効果測定が可能 |

この戦略は「コンビニの壁面・レジ前空間を広告メディアとして収益化する」という発想の転換であり、Google Cloud活用のセブンや KDDI技術活用のローソンとは異なるDXの方向性を示している。

### 4-4. 店舗DX施策

| 施策 | 状況（2024〜2026年） |
|------|------------------|
| **セルフ会計専用レーン** | 全店舗への展開を推進。フルセルフ型ではなくセミセルフ型（スタッフ常駐）が主流 |
| **AI発注（廃棄削減）** | Azure AIを活用した発注最適化。食品廃棄削減を中期目標の一つに明記 |
| **ファミペイ（決済・ポイント）** | 2,000万DL超。Tポイント提携終了後にファミペイ独自経済圏を構築 |
| **デジタルサイネージ広告** | 国内コンビニで最も積極的。店内サイネージを収益源に転換 |
| **伊藤忠データ連携** | ユニー（GMS）・繊維・食料部門のサプライチェーンデータを活用した商品政策 |
| **FC加盟店支援デジタル化** | SV業務のDX（タブレットベースの指導・数値共有）。加盟店との情報格差解消 |

### 4-5. 競合との差別化ポイント

- **メディア事業の先進性**: 店頭サイネージ広告のマネタイズはコンビニ3社の中で最も進捗
- **伊藤忠グループの商社機能**: 海外調達・サプライチェーン最適化でメーカーとの交渉力が高い
- **Azure OpenAI の早期活用**: 国内小売業の中でも先進的なOpenAI技術の適用（2023〜2024年）
- **非上場による長期投資**: 2020年の非上場化以降、四半期プレッシャーなくDX投資を継続可能

## 5. ローソンのDX戦略（詳細）

### 5-1. 経営体制の変化（2024年上場廃止の意味）

2024年7月のローソン上場廃止は、DX戦略を理解する上で最も重要な背景情報である。

| 変化 | 内容 | DXへの影響 |
|------|------|---------|
| **KDDI（50%）+ 三菱商事（50%）の完全支配** | 少数株主を排除し、2社で100%保有 | 両社の意思決定が直接ローソンのDX方針に反映 |
| **上場廃止** | 四半期決算・短期株価プレッシャーから解放 | ROI回収まで時間を要するDX投資（クラウド基盤・ストコン刷新）に踏み切れる |
| **KDDIのテクノロジー注入** | WAKONX基盤・AI・ネットワーク・EdgeAI | 通信キャリアのインフラ技術をコンビニ運営に直接適用 |
| **三菱商事のサプライチェーン** | 食品・資源・物流ノウハウ | メーカー交渉・物流最適化・海外展開での競争力 |

### 5-2. LDI（ローソンデジタルイノベーション）の役割

LDIは「コンビニIT内製化」の最先進事例として業界内で注目されている。

| 項目 | 内容 |
|------|------|
| 設立 | 2016年1月（ローソン66%・シグマクシス34%の共同出資） |
| 完全子会社化 | 2020年4月 |
| 従業員数 | 約123名（2025年時点） |
| 主な開発領域 | ローソンアプリ（2,000万DL）、Webフロントエンド、サーバーサイド、データサイエンス |
| 技術スタック | iOS（Swift）、Android（Kotlin）、サーバー（Go/Gin）、Web（Next.js/AWS Amplify）、クラウド（AWS中心） |
| 組織体制 | コンシューマーサービス部・インフラ部門・データサイエンス部門・品質管理部門・セキュリティ部門 |

LDIが担うのはコンシューマー向けアプリ・Web・データ分析領域であり、ストコンや POS などの「現場系システム」は引き続き NEC・富士通・NTTデータなどの外部ベンダーが担当している点に注意が必要。

### 5-3. Real×Tech LAWSON 構想

2025年6月に高輪ゲートウェイシティでオープンした「Real×Tech LAWSON」は、ローソンのDXビジョンを凝縮した実験店舗である。

**搭載した15のリテールテック**:

| カテゴリ | テクノロジー | 概要 |
|---------|-----------|------|
| **顧客体験** | AIサイネージ | AIカメラで顧客行動解析。関連商品をリアルタイム推奨。通常バナー比**約4倍の閲覧効果** |
| **顧客体験** | AI Ponta | 生成AIロボットが来店者にレコメンド・周辺情報提供 |
| **顧客体験** | スマートシティ連携 | KDDI都市OSから天気・電車遅延・混雑情報をリアルタイム表示 |
| **顧客体験** | 3Dアバター遠隔接客 | Pontaよろず相談所で健康・金融相談を遠隔対応 |
| **省人化** | 自動調理ロボ | からあげクン等の調理を自動化 |
| **省人化** | 飲料陳列ロボ | ペットボトル等の棚補充を自動化 |
| **省人化** | 自動掃除ロボ | 店内清掃の自動化 |
| **店舗管理** | スマートシェルフ | デジタル在庫棚でリアルタイム在庫管理 |
| **店舗管理** | AIエージェント分析 | 防犯カメラ映像から業務量を算出、課題抽出〜改善提案 |

**WAKONX Retail プラットフォームとの関係**: Real×Tech LAWSON はKDDIの WAKONX Retail（リテール向けAIプラットフォーム）の実装ショーケースとして機能する。KDDI にとってもローソン以外の小売業への横展開（他社への提案実績）として活用。

### 5-4. クラウド基盤の詳細（マルチクラウド環境）

ローソンは単一クラウドではなく、用途別に4種のクラウド/基盤を並存運用している。

| クラウド/基盤 | 用途 | 主要パートナー | 特徴 |
|------------|------|-------------|------|
| **AWS** | 新規開発システム全般（アプリバックエンド・LDIシステム） | 日立ソリューションズ、アイレット、NEC | LDIが主体で開発。Next.js、Go、DynamoDB等を使用 |
| **NSSOL absonne** | 既存業務システム（リフト&シフト対象） | NSSOL（新日鉄住金ソリューションズ） | 既存ストコン関連システムを含む旧来の基盤 |
| **Azure** | IoT在庫管理実証（RFID）・店舗DX（カメラAI） | Microsoft | 実証フェーズ段階。本番化は一部 |
| **KDDI WAKONX** | AIサービス基盤・データ分析・会員統合（auとPonta連携） | KDDI | Real×Tech LAWSON の基盤。Ponta×au会員データ統合 |

**ストコン AWS移行案件との接点**:
- 現在 NSSOL absonne 上にある既存ストコン関連システムを **AWS に移行**するのが本案件の核心
- 移行後は、現場（NEC製ストコン）→ AWS（新基盤）→ KDDI WAKONX（AI分析）→ LDI（アプリ）という新しいデータフローが確立される

### 5-5. AI.CO（発注AI）の全店展開

| 項目 | 内容 |
|------|------|
| 開発元 | フューチャーアーキテクト（ITコンサル・開発会社） |
| 機能 | 発注支援AI（品揃え・値引き・廃棄削減を一体化） |
| 全店展開完了 | **2024年7月** |
| 対象店舗 | 全約14,500〜14,700店 |

## 6. トライアルのDX戦略（参考）

トライアルはコンビニではなくスーパーセンター業態だが、コンビニと競合する小型フォーマット「トライアルGO」の展開と、業界で最も進んだ「AI・IoT活用型店舗」として参考になる。

### 6-1. RetailAI の取り組み

トライアルグループの100%子会社である「RetailAI（リテールAI）」は、AI・データ技術を核とした小売DX推進会社。

| 技術 | 内容 |
|------|------|
| **AIカート** | 買い物カートにタブレット・カメラ・バーコードリーダーを搭載。商品スキャン→自動精算（カートレジ）。顧客行動データのリアルタイム収集 |
| **AI棚割** | AIカートの購買行動データを活用した棚割最適化 |
| **デジタルサイネージ** | AIカートの画面に広告・レコメンドを配信。購買文脈に合わせた精度の高い広告 |
| **トライアルGO（小型業態）** | セルフレジ・AIカートを標準搭載した「人手最小化店舗」モデル |
| **データ販売** | RetailAI が蓄積した購買行動データをメーカーに販売。小売DXの新収益モデル |

### 6-2. コンビニ業界へのインプリケーション

| コンビニとの対比 | 示唆 |
|--------------|------|
| AIカートは「顧客をデータソース化」する仕組み | コンビニのセルフレジ×アプリ連携も同じ方向性。購買行動データの精度が上がる |
| 購買行動データのメーカー販売 | セブン・ファミマもPOSデータ販売を展開するが、行動データの粒度ではトライアルが先行 |
| 人手最小化の徹底 | コンビニの「ゼロアシスト店舗」実験の参考モデル |
