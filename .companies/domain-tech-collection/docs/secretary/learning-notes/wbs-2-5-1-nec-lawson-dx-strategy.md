## 1. ローソン経営体制の変化

### 1-1. KDDI + 三菱商事 + LDI の三者構造

2024年に起きた経営体制の変化は、ローソンのDX戦略の方向性を根本から変えた。

| プレイヤー | 出資・立場 | 主な役割 |
|-----------|-----------|---------|
| **KDDI** | 議決権 50%（三菱商事と折半） | 通信基盤・会員データ連携・「Real×Tech Lawson」推進・**WAKONX** プラットフォーム |
| **三菱商事** | 議決権 50%（KDDI と折半） | サプライチェーン・物流ノウハウ・産業 DX プラットフォーム |
| **ローソンデジタルイノベーション（LDI）** | ローソン×シグマクシスの共同出資 IT 戦略子会社 | **IT 内製化の中核**。デジタル施策の企画・推進 |
| **NEC** | 外部ベンダー | 映像認識 AI・LLM 実証・顔認証技術・SI 案件 |

### 1-2. 上場廃止の意味

- **2024年7月: 上場廃止** → 四半期ごとの株価プレッシャーから解放
- 短期的な株価配慮なしに、中長期の DX 投資を断行できる経営体制へ
- KDDI・三菱商事の2社が意思決定を完全に握るため、**意思決定が速く、大型投資を実行しやすい**

**PM として知っておくべきポイント**
上場廃止で「ROI が出るまで待てる体制」が整った。NEC が提案するシステム刷新（ストコン AWS 移行を含む）も、短期 ROI だけでなく**中長期のスケーラビリティ・信頼性**で訴求できる。逆に言えば、KDDI・三菱商事の経営判断が最終的なゴーサインを握るため、両社が納得するビジネスケースが必要になる。

## 2. NEC のポジション

### 2-1. NEC が担う守備範囲

ローソン DX 全体の中で NEC が受け持つ領域は限定的だが、技術的な深度は高い。

| 領域 | NEC の関与 | 主な技術 |
|------|-----------|---------|
| **映像認識 AI + LLM** | 2025年10月〜11月 実証実験（埼玉 1 店舗） | 映像分析基盤（EVA）、LLM による作業工程の文章化 |
| **顔認証技術** | セブン-イレブン向けで実績あり（約40万人規模）。ローソン向けは今後 | NEC 独自の顔認証エンジン |
| **SI（システムインテグレーション）** | ストコン（店舗コンピューター）周辺の基盤系 | オンプレ→クラウド移行等 |
| **発注支援 AI** | セブン-イレブン向けでは発注時間 35%削減・欠品率 27%改善の実績 | 「説明できる AI」（判断根拠を可視化） |

### 2-2. KDDI / LDI との境界線

```
[ローソン DX の全体構造]

KDDI 領域                LDI 領域               NEC 領域
─────────────────────────────────────────────────────────
通信インフラ             IT 戦略企画            映像認識 AI
WAKONX 基盤             内製アプリ開発          顔認証
会員データ統合           AI.CO 運用管理          SI 案件
Real×Tech Lawson 推進   DX ロードマップ策定      ストコン周辺基盤
```

- **LDI（ローソンデジタルイノベーション）** は IT 内製化の中核であり、NEC とは補完関係にある
- **NEC の強み**は「既存店舗インフラを熟知していること」と「映像 AI・顔認証等の独自技術」
- **KDDI の強み**は通信基盤と顧客データ。この領域に NEC が入り込むのは難しい

**PM として知っておくべきポイント**
NEC の守備範囲は「既存インフラの刷新（ストコン等）」と「映像・認証系の専門 AI 技術」に集中している。**ストコン AWS 移行は NEC の主戦場**であり、LDI が内製化しようとする領域とは競合しにくい。ただし LDI との連携姿勢（API 連携・データ共有設計）を明示しないと関係が悪化するリスクがある。

## 3. Real×Tech Lawson 構想

### 3-1. 構想の概要

「Real×Tech Lawson」はローソンが掲げる次世代コンビニのビジョン。KDDI の **WAKONX** と連動する形で、デジタルとリアル店舗を融合させる。

| DX 施策 | 概要 | 現状 |
|---------|------|------|
| **AI.CO（AI Customized Order）** | 個店別発注・値引き推奨・廃棄削減を実現するAI発注システム | **全店導入済み** |
| **店舗運営支援 AI** | 天井カメラ 8〜12 台で顧客行動追跡・個店施策提案 | 実証段階 |
| **セルフレジ** | フルオート展開（大手コンビニ唯一） | **4,500 店舗以上** |
| **アバター接客「AVACOM」** | 無人店舗運営 | 一部導入 |
| **ウォークスルー決済** | 東急電鉄二子玉川駅内で実装 | 特定拠点のみ |
| **KDDI 連携顧客データ基盤** | KDDI + ローソン会員情報統合 → 国内最大級の顧客データ基盤 | 構築中 |

### 3-2. NEC はここにどう絡むか

- **映像認識 AI（EVA）**を Real×Tech の「店舗オペレーション効率化レイヤー」に組み込む役割
- 2025年10月の実証では「**従業員作業 30% 削減**」を目標に、映像 AI + LLM で自動レポート生成
- 将来的には**顧客行動分析（来店客の動線ヒートマップ・棚割り最適化）**への拡張が見込まれる
- **ストコン（店舗コンピューター）** は Real×Tech の根幹インフラ。AWS 移行によりクラウドネイティブな DX 施策（AI.CO のデータ連携強化等）との接続が可能になる

**PM として知っておくべきポイント**
ストコン AWS 移行は「単なるインフラ更新」ではなく、**Real×Tech Lawson のデジタル基盤強化の一部**として位置付けられる。移行後のクラウド基盤が AI.CO やKDDI データ基盤との連携強化につながることを PM として語れるようにしておく必要がある。

## 4. ストコン移行案件への示唆

### 4-1. DX 戦略の中でのストコン移行の位置付け

```
[ローソン DX ロードマップとストコン移行の関係]

KDDI / LDI 主導レイヤー
  ├── 会員データ基盤（WAKONX）
  ├── AI.CO（発注最適化）
  └── Real×Tech 顧客体験

                  ↑ データ連携・API 連携
                  │
NEC 主導レイヤー（ストコン移行がここに該当）
  ├── ストアコンピューター（店舗の基幹システム）
  ├── POS 連携基盤
  └── 店舗オペレーション（映像 AI・顔認証）
```

- **ストコン移行はローソン DX の「土台」にあたる** — ここが不安定だと上位の AI 施策も動かない
- オンプレ→AWS 移行によって、クラウドネイティブな API 連携（LDI が開発するアプリとのデータ接続）が実現する
- **2026年6月参画時点では Real×Tech 構想がさらに具体化している**と想定。NEC 社内でどこまで情報共有されているか確認が必要

### 4-2. NEC の立ち回りの注意点

| リスク | 内容 | 対策 |
|--------|------|------|
| **LDI との役割分担の曖昧さ** | LDI が内製化を進める中、NEC の担当範囲が侵食される可能性 | 契約・スコープの明確化を参画初日から実施 |
| **KDDI 主導の技術選定変更** | KDDI が独自クラウド基盤を推進した場合、AWS 選定が見直されるリスク | ステークホルダー分析で KDDI 側の意向を早期確認 |
| **映像 AI 実証との競合優先度** | 映像 AI・LLM 案件が拡大するとストコン移行のリソースが削られる可能性 | NEC 社内リソース計画を PM として早めに把握 |
| **セブン-イレブン先行案件との比較** | セブンが Google Cloud + マイクロサービスで先行。ローソン側が「なぜ AWS か」を問う可能性 | AWS 選定の技術的根拠を明確化（既存インフラとの親和性等） |

**PM として知っておくべきポイント**
2026年6月参画後、最初の 1 か月で「**この移行案件が Real×Tech ロードマップのどのマイルストーンに対応しているか**」を LDI・ローソン IT 担当・KDDI 担当と合意しておく。位置付けが曖昧なまま進むと、途中でスコープ変更やリソース競合が起きやすい。

## 5. キーパーソンと意思決定構造

### 5-1. ステークホルダー整理

レポートから読み取れる主なステークホルダーを整理する。

| 所属 | 立場・役割 | ストコン案件への影響度 |
|------|-----------|-------------------|
| **ローソン / KDDI 側** | | |
| KDDI 経営層 | Real×Tech Lawson の最終意思決定者。DX 投資のゴーサインを出す | 高（投資承認） |
| 三菱商事経営層 | サプライチェーン・物流観点での意思決定者 | 中（間接的に関与） |
| **LDI（ローソンデジタルイノベーション）** | IT 戦略の企画・内製化推進。NEC との窓口になる可能性大 | 高（技術要件・スコープの定義） |
| **NEC 側** | | |
| NEC 営業・SE（ローソン担当） | 案件の窓口。提案〜契約をリード | 高（直接の交渉相手） |
| NEC 映像 AI チーム | 映像認識 AI・LLM 実証の担当。ストコン担当とは別部署の可能性 | 中（将来的な連携機会） |
| **PM（笹尾）** | 2026年6月から NEC 側 PM として参画 | ─（自分自身） |

### 5-2. 意思決定の流れ（想定）

```
[意思決定フロー]

KDDI / 三菱商事（経営層）
       ↓ DX 投資方針
LDI（IT 戦略策定・内製化）
       ↓ 要件・スコープ定義
NEC 営業・SE
       ↓ 提案・設計・実装
PM（笹尾）← ストコン移行の現場をここが回す
```

**PM として知っておくべきポイント**
意思決定の実質的な窓口は **LDI** になる可能性が高い。LDI はシグマクシスとローソンの合弁子会社であり、コンサル出身のメンバーが多い。彼らが「なぜこのアーキテクチャか」「なぜ NEC か」を論理的に問い詰める文化を持つ可能性がある。PM として意思決定の根拠を常にドキュメント化しておく（ADR 形式を活用）。

## 7. まとめ

### 7-1. 今回の学習で得た核心的な理解

1. **ローソン DX の意思決定は KDDI・三菱商事が握る**
   上場廃止により、中長期視点の大型 DX 投資が可能になった。NEC はこの枠組みの中で、既存インフラ刷新と専門 AI 技術提供の役割を担う外部ベンダーである。

2. **NEC の主戦場はストコン周辺と映像 AI・顔認証**
   LDI が担う内製化（アプリ・AI.CO 運用）とは棲み分けができており、**ストコン AWS 移行は NEC の守備範囲内の案件**である。ただし LDI との連携設計を怠ると関係悪化につながる。

3. **ストコン移行は Real×Tech の「土台」である**
   単なるインフラ更新ではなく、AI.CO・KDDI データ基盤・映像 AI との連携強化を可能にする「デジタル基盤の近代化」として位置付けられる。PM として、この上位文脈を語れることがステークホルダー信頼獲得につながる。

### 7-2. 参画前に調べておくべきこと

| 調査項目 | 理由 | 優先度 |
|---------|------|-------|
| **LDI の組織構成と NEC との契約形態** | 窓口・意思決定ルートを把握するため | 高 |
| **現行ストコンのアーキテクチャ詳細** | 移行スコープ・リスクを正確に把握するため | 高 |
| **AWS 選定の経緯と根拠** | 「なぜ AWS か」を問われた際に答えられるようにするため | 高 |
| **映像 AI 実証結果（2025年10月〜11月）** | NEC の提案実績として活用できるため | 中 |
| **UCF（用語確認）** | 業界固有用語として登場する可能性があるため | 中 |
