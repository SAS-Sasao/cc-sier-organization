## Part 1: ストコンとは（1枚サマリー）

### 定義（3行以内）

ストアコンピューター（ストコン）とは、コンビニ各店舗に設置される店舗情報管理システムの中核専用機である。
店長・従業員が日常的に操作する業務端末として、発注・在庫・精算・本部通信のすべてを一元管理する。
店舗内の各システム（POS・マルチメディア端末等）からデータを集約し、本部との双方向通信のハブとして機能する。

### 何をしているか

- **発注管理**: 推奨発注数の算出・確認・本部への発注データ送信（EOS）
- **在庫管理**: 入荷・廃棄・販売を反映した理論在庫の常時管理、棚卸データの突合
- **売上精算**: 日次精算表（日計）の作成、現金実査結果の突合、本部への精算データ送信（EOB）
- **商品マスタ管理**: 本部から受信した商品マスタ・価格・棚割（プラノグラム）を店舗内へ配信
- **サービス連携**: 公共料金収納代行・チケット発券（マルチメディア端末）の売上計上・精算
- **従業員管理**: シフト作成・打刻記録・労働時間集計

### コンビニ業界における位置づけ

| 指標 | 数値（2024年度） |
|------|----------------|
| 業界全体の売上高 | 約12兆円（百貨店全体の約2倍） |
| 国内総店舗数 | 約55,000〜56,000店 |
| 年間来客数 | 約180億人（国民1人あたり年約140回） |
| 上位3チェーンのシェア | セブン/ファミマ/ローソンで約95%の寡占構造 |

> ストコンは約55,000店すべてに設置される、コンビニ店舗オペレーションの"神経中枢"である。
> 本部の戦略・商品・価格・分析知見は、すべてストコンを経由して店舗に届く。

**ポイント**: ストコンは「店舗と本部を結ぶハブ」として機能する。店舗内の全システムからデータを集約して本部へ送信し、本部からの商品マスタ・価格・キャンペーン情報を受信して各端末へ配信する。この双方向ハブ機能がストコンの最大の存在意義である。

### 上り（店舗→本部）の詳細

```
POS レジ
  │
  │ [リアルタイム] 1取引ごとに売上データ送信
  │
  ↓
ストアコンピューター（集約・加工）
  │
  ├── [発注締め時刻] ──→ 発注データ ──────→ 発注管理システム（EOS）
  │       ↑                              ※ 1日2〜3回（朝8時/昼12時/夜20〜22時）
  │       └─ 朝便: 前日20〜22時締め
  │          昼便: 当日6〜8時締め
  │          夕便: 当日10〜12時締め
  │
  ├── [日次: 22:00〜翌2:00 EOB処理] ──→ 精算データ ───→ 会計・ロイヤリティシステム
  │                                  ──→ 廃棄データ ───→ 在庫・棚割管理システム
  │                                  ──→ 勤怠データ ───→ 人件費管理システム
  │
  ├── [入荷時随時] ──→ 検品データ ──────→ 物流管理システム
  │
  ├── [リアルタイム〜日次] ──→ POSデータ ─→ データウェアハウス（DWH）
  │                                          ※ セブンはほぼリアルタイム送信
  │
  └── [棚卸実施時] ──→ 棚卸データ ─────→ 在庫管理システム
```

### 下り（本部→店舗）の詳細

```
本部各システム
  │
  ├── [随時] 商品マスタ ────────────→ ストコン → POSレジへ配信
  │           （新商品登録・廃番・価格改定）
  │
  ├── [随時] 価格変更 ─────────────→ ストコン → POSレジへ配信
  │
  ├── [キャンペーン前] キャンペーン情報 → ストコン → POSレジ・マルチメディア端末
  │           （割引設定・バナー表示等）
  │
  ├── [棚替え前] 棚割マスタ（プラノグラム）→ ストコン（店舗での棚割実施管理）
  │
  ├── [随時] 推奨発注データ ─────────→ ストコン（翌回の発注画面に反映）
  │           ※ 需要予測AI が前日の全店POS・天気・イベント情報から算出
  │
  ├── [随時] 発注制限情報 ─────────→ ストコン（欠品/過剰在庫時の数量上限）
  │
  └── [随時] セキュリティパッチ ─────→ ストコン・POSへ自動適用
```

### 店舗内のデータフロー

```
POSレジ
  │ [リアルタイム: 1取引ごと]
  │ 売上データ（商品コード・数量・支払種別・時刻）
  ↓
ストコン ← [随時] ハンディターミナル（検品・棚卸スキャンデータ）
  ↓
マルチメディア端末（Loppi / Famiポート）
  │ [随時]
  │ チケット販売・収納代行の取引データ
  └──────────────────────→ ストコンへ連携（売上計上）
```

### 外部連携データフロー

```
収納代行（公共料金）
  1. 顧客がバーコード付き払込票を持参
  2. POSでバーコードスキャン → 金額確認 → 現金受取 → 領収書発行
  3. [日次 EOB処理時] 当日の収納代行金額を本部へ報告
  4. [翌日以降] 本部が収納機関（電力会社・税務署等）へ送金

チケット発券（イープラス等）
  1. 顧客がマルチメディア端末で選択・購入
  2. [リアルタイム] 発券枚数データをストコンへ連携
  3. [日次] 売上データを本部へ報告
  ※ イベント当日の大量発券集中対応が処理能力上の重要要件
```

### 主要チェーンで異なる点（注記）

| 項目 | セブン-イレブン | ファミリーマート | ローソン |
|------|--------------|----------------|--------|
| POSデータ送信タイミング | ほぼリアルタイム（業界最速クラス） | リアルタイム〜バッチ | バッチ中心（随時化進行中） |
| 推奨発注の精度 | AI発注が最も進化（独自AI）| Azure AI活用で高度化中 | KDDI連携でスマート化推進 |
| マルチメディア端末連携 | セブン-イレブン端末（要確認） | Famiポート | Loppi |
| 下りデータ通信の特徴 | SIS（セブン情報システム）経由 | ファミマIT本部主導 | KDDIネットワーク活用検討中 |

> 注意: 各チェーンの詳細なシステム仕様は非公開情報が多い。上記は公開情報・業界慣行に基づく一般的情報。案件参画後はNDAの範囲で確認すること。

### SIer参入の視点（PM向け補足）

コンビニITの主要領域は大手ベンダー（NEC・富士通・NTTデータ・日本IBM）が抑えている。
中堅SIerが入りやすいニッチ領域は以下の通り。

- クラウド移行PMO（オンプレからAWS/Azureへの移行プロジェクト管理）
- AI発注基盤の新規構築・改善（データ整備・MLOps）
- エッジIoT連携（温度管理・電子棚札・カメラ解析）
- PCI DSS対応・セキュリティ強化
- EC・デリバリー基盤の新規構築

既存ベンダーとのアライアンス・サブコン参入が現実的なアプローチ。
