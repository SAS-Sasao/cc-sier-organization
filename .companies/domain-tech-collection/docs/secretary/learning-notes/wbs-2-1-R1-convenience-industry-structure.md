## 1. 日本のコンビニ業界の全体構造

### 1-1. 市場規模と店舗数

日本のコンビニエンスストア業界は、小売業の中でも突出した存在感を持つ。

| 指標 | 数値（2024年度） | 備考 |
|------|-----------------|------|
| 業界全体の売上高 | 約12兆円（チェーン合計） | 百貨店全体（約5.9兆円）の約2倍 |
| 国内総店舗数 | 約55,000〜56,000店 | ピーク時（2019年）は約58,000店 |
| 年間来客数 | 約180億人 | 日本国民1人当たり年約140回 |
| 平均客単価 | 約700〜750円 | キャッシュレス普及で上昇傾向 |
| 店舗あたり日販（日次売上） | 約50〜70万円 | チェーン・立地により大きく差異 |

**市場の特徴**:
- 総店舗数は2019年をピークに微減傾向（人手不足・競合激化）
- 売上高は横ばいから微増。商品単価上昇・サービス収入増が補完
- 1都3県（東京・神奈川・埼玉・千葉）に全国店舗の約20%が集中

### 1-2. 主要チェーンの勢力図

| チェーン | 運営会社 | 親会社グループ | 国内店舗数（2024年） | 海外展開 |
|---------|---------|--------------|-------------------|---------|
| セブン-イレブン | セブン-イレブン・ジャパン | セブン&アイ・ホールディングス | 約21,500店 | 米国・タイ・韓国等19カ国/地域 |
| ファミリーマート | ファミリーマート | 伊藤忠商事（主要株主） | 約16,500店 | 中国・台湾・東南アジア等 |
| ローソン | ローソン | 三菱商事・KDDI（各50%出資） | 約14,500店 | 中国・東南アジア等 |
| ミニストップ | ミニストップ | イオングループ | 約1,900店 | 韓国・東南アジア等 |
| セイコーマート | セコマ | セコマグループ | 約1,200店（北海道中心） | なし |

**業界構造の特徴**:
- 上位3チェーン（セブン・ファミマ・ローソン）で全体の約95%を占める**寡占構造**
- セブン-イレブンが売上・利益ともに圧倒的首位を維持（日販差が約10〜15万円）
- ローソンは2024年2月に三菱商事・KDDIによる完全子会社化（TOB完了）、KDDIとのデジタル連携強化へ
- ファミリーマートはサークルKサンクス統合（2016年）後、店舗品質の均質化に注力

### 1-3. 業界全体の収益構造

コンビニ本部の収益は、加盟店から受け取る**チャージ（ロイヤリティ）**が主軸。

```
本部収益の主な構成要素:
1. フランチャイズ・チャージ          → 加盟店粗利益の一定率（後述）
2. 金融・決済手数料収入             → 収納代行、ATM利用料、電子マネー
3. 物流・共配センター運営費         → 加盟店からの配送手数料
4. 商品開発・プライベートブランド    → 商品差別化と利益率向上
5. デジタル・メディア収入           → アプリ広告、ポイント経済圏
```

## 2. フランチャイズモデルの仕組み

### 2-1. 本部-加盟店関係の基本構造

コンビニのFCモデルは、**本部がシステムと商品力を提供し、加盟店がオペレーションを担う**分業構造。

```
[本部（チェーン本部）が提供するもの]
- 商品（特にPB商品・デイリー品の独占供給）
- システム（POS・ストコン・発注システム一式）
- 物流（共配センター・配送網）
- 広告・マーケティング
- SVによる経営指導
- 加盟店への融資・保証

[加盟店（オーナー）が負担するもの]
- 土地・建物費用（タイプAはオーナー所有、タイプCは本部借り上げ）
- 従業員人件費
- 光熱費
- チャージ（ロイヤリティ）の支払い
- 廃棄ロスの一部負担
```

### 2-2. チャージ（ロイヤリティ）の構造

チャージは「粗利益分配方式」が基本。売上ではなく**粗利益に対する一定率**を本部に支払う。

| チェーン | チャージ率（粗利基準） | 備考 |
|---------|---------------------|------|
| セブン-イレブン | 約43〜76%（タイプ別） | タイプC（建物本部借り上げ）は最高76% |
| ファミリーマート | 約36〜64%（タイプ別） | 開業支援の充実度により差異 |
| ローソン | 約40〜72%（タイプ別） | エリア・立地条件で変動 |

**粗利益からチャージを引いた残りが加盟店の取り分（オーナー収益）**。そこから人件費・光熱費等を払う。
廃棄ロス（売れ残りの廃棄）は加盟店の粗利益を直接減らすため、**廃棄ロス削減が加盟店の最重要課題**の一つ。

### 2-3. 加盟店オーナーの経済的実態（PM理解として重要）

- 年間営業時間：24時間×365日 = 8,760時間
- 1店舗あたりの年間人件費：約3,000〜5,000万円（アルバイト含む）
- 加盟店オーナーの手取り収入：年間300〜700万円程度（立地・経営力で大差）
- **24時間営業問題**: 2019年以降、本部と加盟店オーナーの関係が社会問題化。時短営業実証実験が各チェーンで拡大

**システム導入判断への含意**: 加盟店の「廃棄ロス削減」「人件費削減」に直結するシステム改善は、加盟店の協力を得やすい。逆に本部都合のシステム変更（オーナー負担増）は現場抵抗を生む。

### 2-4. スーパーバイザー（SV）の役割

SVはコンビニFCの重要な仕組みの一つ。1人のSVが担当する店舗数は平均7〜12店。

| SVの主な業務 | 内容 |
|------------|------|
| 経営指導 | 売上改善・発注精度向上・廃棄削減のアドバイス |
| 店舗チェック | 清潔さ・陳列・スタッフ対応の確認 |
| 新商品・キャンペーン展開 | 本部施策の現場への橋渡し |
| トラブル対応 | クレーム・緊急事態の一次対応 |
| データ分析支援 | ストコン活用・単品管理の指導 |

SVは**店舗とデジタルの接点**でもある。ストコンのデータを見ながら発注アドバイスをするため、ストコンのUI/UX改善はSVの業務効率に直結する。

## 3. サプライチェーン構造

### 3-1. コンビニの物流体制の全体像

コンビニのサプライチェーンは**温度帯別共同配送**が最大の特徴。

```
[メーカー・ベンダー]
    ↓ (納品)
[共配センター（温度帯別）]
  ├── 常温センター（加工食品・日用品・文具等）
  ├── チルドセンター（弁当・惣菜・牛乳・デザート：5〜8℃）
  ├── 冷凍センター（アイス・冷凍食品：-25℃）
  └── 米飯センター（おにぎり・サンドウィッチ：20℃前後）
    ↓ (日3便〜週1便)
[店舗]
```

**温度帯と配送頻度**:

| 温度帯 | 主な商品 | 1日の配送回数 |
|-------|---------|-------------|
| チルド（デイリー品） | 弁当・おにぎり・サンド・惣菜・牛乳 | 日3便（朝・昼・夕） |
| 米飯（常温品の一部） | おにぎり単体（品質管理の厳格さから分離） | 日2〜3便 |
| 常温 | 加工食品・スナック・飲料 | 週2〜3便 |
| 冷凍 | アイス・冷食 | 週1〜2便 |
| 雑貨・文具 | 日用品・雑誌・タバコ | 週1〜2便 |

### 3-2. 共配センター（DC）の仕組み

コンビニの共配センターは、複数メーカーの商品を一括して各店舗へ配送する**クロスドッキング型物流拠点**。

- **オーナーシップ**: 本部が設立・管理する専用センターが主流。物流子会社が運営（例：セブン-イレブンの場合、株式会社セブン-イレブン・ロジスティクス）
- **センター規模**: 1センターが担当するエリアの店舗数は50〜300店
- **在庫レス設計**: 基本的にセンターには在庫を持たない（翌日には次便で商品が流れる設計）
- **メーカーへの影響力**: 大量取引を背景に、メーカーに対して**商品規格・包装・賞味期限の指定権**を持つ

### 3-3. 商品の需給調整メカニズム

```
[発注の流れ]
店舗（ストコン経由で発注） → 本部発注システム → 共配センター → 店舗へ納品

[情報の流れ（逆方向）]
売上実績（POS）→ 本部DWH → 需要予測AI → 推奨発注数算出 → ストコン表示
```

**単品管理（単管）**: コンビニ独自の概念。すべての商品を単品レベルで発注・管理する。品切れも廃棄ロスも同程度に「機会損失・損失」と捉え、最適在庫を追求する考え方。

### 3-4. 返品・廃棄の仕組み

| 商品区分 | 廃棄のタイミング | 廃棄コスト負担 |
|---------|---------------|-------------|
| デイリー品（弁当等） | 消費期限直前（独自の販売期限設定） | 基本的に加盟店（粗利減少） |
| 雑誌・書籍 | 発売後2〜4週間 | メーカーへの返品が可能（委託販売） |
| 日用品・食品（長期） | 一定期間後 | 加盟店負担 |

**2019年以降の値引き拡大**: 消費期限前の値引き販売（いわゆる「チキンレース販売」）を各チェーンが解禁・拡大。加盟店の廃棄ロス削減と食品ロス法対応を両立する取り組み。

## 4. コンビニの業務プロセス全体像

### 4-1. 店舗運営の1日の流れ

```
[深夜帯] 00:00〜06:00
 - 夜間スタッフによる補充・清掃
 - デイリー品（弁当等）の廃棄処理・ストコン入力
 - 深夜便の納品受け入れ・検品
 - 翌朝便の発注確認（ストコン）

[朝帯] 06:00〜11:00
 - 朝一番の納品受け入れ（最も商品量が多い）
 - 弁当・おにぎりの品出し・陳列
 - 日次精算（前日分のレジ締め・ストコン日次処理）
 - 店長・SVとのデータ確認

[昼帯] 11:00〜14:00
 - 昼のピーク対応（最多来客時間帯）
 - 昼便の納品受け入れ
 - 在庫確認・欠品対応・追加発注

[午後帯] 14:00〜18:00
 - 商品の品出し・陳列整理
 - 棚割変更・プロモーション展開
 - 夕方便の納品準備

[夕方〜夜帯] 18:00〜00:00
 - 夕食需要ピーク対応
 - 夕便の納品受け入れ
 - 翌日分の発注作業（ストコン操作）— 1日の業務の中核
 - 閉店前の廃棄処理
```

**発注作業の重要性**: 1日の業務の中で最も重要かつ技能を要するのが**発注（推奨発注の調整）**。ストコンの推奨発注数をベースに、天気・地域イベント・近隣競合状況を勘案して手動調整する。熟練オーナーとそうでないオーナーで廃棄ロスに2〜3倍の差が出ることもある。

### 4-2. 商品カテゴリ別の特性

| カテゴリ | 代表商品 | 特性 | システム要件 |
|---------|---------|------|------------|
| **デイリー品（FF品）** | 弁当・おにぎり・サンドウィッチ・パン | 消費期限が短い（当日〜翌日）、1日3便納品、廃棄リスク高 | リアルタイム在庫・精緻な需要予測・廃棄自動入力 |
| **チルド加工食品** | 牛乳・ヨーグルト・チーズ・デザート | 賞味期限1週間前後、チルド配送必須 | 温度帯管理・先入先出管理 |
| **加工食品（常温）** | カップ麺・スナック・菓子・缶詰 | 長期保管可能、季節性あり | 棚割管理・プロモーション連携 |
| **飲料** | ペットボトル・缶・コーヒー | 大量販売、季節性強い | 気温連動発注・大量在庫管理 |
| **タバコ** | 紙巻・電子タバコ（IQOS等） | 高単価・免許品、在庫管理厳格 | 規制対応・年齢確認連動 |
| **日用品・雑貨** | 衛生用品・文具・乾電池 | 緊急需要（購入動機が衝動的）、季節性あり | 棚割最適化 |
| **サービス** | 宅配便・公共料金・チケット | 在庫レス、手数料収入、顧客接点 | 外部システム連携（ヤマト・ゆうパック・各収納代行） |
| **カウンターFF** | コーヒー・揚げ物・中華まん | ロスリスク高、機器管理が必要 | IoT機器連携・廃棄管理 |

### 4-3. 本部-店舗間の情報フロー

```
[下り情報（本部 → 店舗）]
 - 商品マスタ（新商品・廃番・価格変更）
 - 棚割プラノグラム
 - キャンペーン・販促情報
 - 推奨発注数（需要予測AI算出値）
 - 業務連絡（緊急リコール、新規サービス案内）
 - システムアップデート

[上り情報（店舗 → 本部）]
 - POS売上データ（商品・時間帯・支払い方法）
 - 発注データ
 - 廃棄データ
 - 在庫データ（棚卸・日次）
 - 精算データ
 - 機器状態（異常アラート）

[通信タイミング]
 - リアルタイム: 決済・POS（ほぼリアルタイム）
 - バッチ（日次）: 売上確定・精算・在庫サマリ
 - 随時: 発注送信・マスタ受信
```

## 5. IT/システム観点での業界構造

### 5-1. 各チェーンのIT投資動向・デジタル戦略（2024年度）

| チェーン | IT投資の重点領域 | デジタル戦略の特徴 |
|---------|---------------|-----------------|
| **セブン-イレブン** | AI発注精度向上・7payの再挑戦（後継サービス）・7-Eleven App | グループ全体（セブン&アイ）のデジタル統合。EC「7NOW」とのオムニチャネル化。生成AIを活用したSV業務支援 |
| **ファミリーマート** | Azure AI活用・ファミペイ経済圏・デジタルサイネージ | 伊藤忠商事グループのデータ活用基盤との連携。店頭サイネージ広告のメディア化（ファミリーマートのメディア事業）。ユニー（GMS）との相互送客 |
| **ローソン** | KDDIとの協業（au PAY連携・ローソンスマホレジ）・三菱商事のサプライチェーン最適化 | ヘルスケア領域（調剤薬局併設）での差別化。スマートストア構想（フルセルフ化実験）。LINEヤフーとの会員連携 |
| **ミニストップ** | イオングループ共通基盤への統合 | WAONポイント連携・イオンモール内店舗展開。ソフトクリームを中核とした差別化戦略 |

### 5-2. 店舗系システムの全体マップと位置づけ

店舗に設置されるシステムは多岐にわたる。それぞれの役割と連携を整理する。

```
[店舗内システム全体図]

┌─────────────────────────────────────────┐
│              店舗システム                  │
│                                         │
│  [ストアコンピューター（ストコン）]           │
│    ↕                                    │
│  [POSレジ]  ←→ [電子マネー/決済端末]      │
│                   [QR決済端末]           │
│    ↕                                    │
│  [マルチメディア端末]                      │
│  （Loppi / Famiポート / Lチキポート等）     │
│    ↕                                    │
│  [ATM]（セブン銀行/ローソン銀行/イーネット）  │
│    ↕                                    │
│  [バックヤード機器]                        │
│  カフェマシン・フライヤー・電子棚札（ESL）    │
└──────────────┬──────────────────────────┘
               │ 回線（専用線/インターネットVPN/5G）
┌──────────────┴──────────────────────────┐
│              本部システム                  │
│  商品マスタ / DWH / 需要予測AI / 経営管理  │
└─────────────────────────────────────────┘
```

| システム | 主な役割 | システム所有者 |
|---------|---------|-------------|
| ストアコンピューター | 発注・在庫管理・精算・マスタ配信ハブ | 本部（チェーン）が提供 |
| POS（レジ） | 会計・売上データ収集 | 本部（チェーン）が提供、メーカーはNEC・東芝TECが主流 |
| マルチメディア端末 | チケット・収納代行・各種サービス | 本部（外部サービス企業と提携） |
| ATM | 現金引き出し・各種決済 | セブン銀行・ローソン銀行が自社設置 |
| 電子マネー/決済端末 | Suica・iD・WAON等の電子マネー決済 | 外部決済事業者と連携 |
| QR決済端末 | PayPay・d払い・au PAY等 | 外部決済事業者と連携 |

### 5-3. 本部系システムの全体像

```
[本部ITシステム全体構成（概念図）]

├── [フロントシステム群]
│   ├── 商品マスタ管理システム（商品DB・棚割システム）
│   ├── 発注管理システム（EOS/EOB）
│   ├── 物流管理システム（WMS・輸配送最適化）
│   └── 加盟店管理システム（契約・精算・SV管理）
│
├── [バックオフィス系]
│   ├── 会計・財務管理（ERPに相当）
│   ├── 人事・給与管理（本部従業員）
│   └── 購買・取引先管理（メーカー・ベンダー管理）
│
├── [情報系・分析系]
│   ├── データウェアハウス（全店POS・在庫データ集約）
│   ├── 需要予測AI（AI/ML基盤）
│   ├── BI・経営ダッシュボード
│   └── 単品分析・マーチャンダイジング支援
│
└── [デジタル・EC系]
    ├── 公式アプリ（顧客向け）
    ├── 会員管理・ポイント基盤（CRM）
    ├── EC・デリバリー基盤（7NOW等）
    └── デジタルサイネージ管理（店頭広告）
```

### 5-4. チェーンとIT子会社・主要ベンダーの関係

| チェーン | IT子会社・関連会社 | 主要ITベンダー |
|---------|----------------|-------------|
| セブン-イレブン | セブン&アイ・ネットメディア、セブン銀行 | NEC（ストコン・POS）、富士通、日立 |
| ファミリーマート | ファミリーマート IT本部（内製化強化） | 富士通（POS）、NEC、TIS（IT基盤） |
| ローソン | ローソンデジタルイノベーション | NTTデータ、富士通、NEC |
| ミニストップ | イオングループIT共通基盤 | NTTデータ（イオングループ共通）、富士通 |

**SIer参入の文脈**: 主要ベンダーとの競合ではなく、**新規領域（クラウド移行PMO・AI発注基盤・エッジIoT・PCI DSS対応）でのニッチ参入**が現実的。既存ベンダーとのアライアンスやサブコン参入が多い。

## 6. 業界課題とトレンド（2024-2026）

### 6-1. 人手不足と省人化

日本のコンビニ業界における最大の構造課題が**人手不足**。

| 指標 | 状況 |
|------|------|
| 1店舗あたりのアルバイト数 | 約15〜20人（常時稼働3〜4名） |
| 時給水準（2024年） | 首都圏1,100〜1,300円（最低賃金上昇で急騰） |
| 外国人労働者比率 | 一部チェーンで30〜40%超の店舗も存在 |
| 人件費の売上比率 | 約20〜25%（加盟店収益を圧迫） |

**省人化の主要施策**:

1. **セルフレジの全店展開（2024〜2026年）**
   - セブン: 「セルフPOS」導入加速。フルセルフ型・セミセルフ型の両方展開
   - ローソン: 「ローソンスマホレジ」（客自身がスマホで会計）。2026年までに主要都市で大幅拡大
   - ファミマ: セルフ会計専用レーンの設置拡大

2. **無人・省人店舗の実験拡大**
   - 深夜帯のみ無人運営（ローソン・大学キャンパス内実証実験等）
   - AI・カメラで万引き検知（顔認証・棚センサー活用）
   - 完全無人店舗はまだ実証段階（2026年時点で商業展開は限定的）

3. **業務自動化**
   - 電子棚札（ESL）による価格変更自動化（大量の棚札張り替え作業を削減）
   - AI発注による発注作業の自動化・軽減
   - ロボットによる清掃・棚補充（一部実験中）

### 6-2. 環境規制・食品ロス削減

2019年施行の**食品ロス削減推進法**をきっかけに、業界全体で食品ロス対応が急加速。

| 施策 | 内容 | 進捗 |
|------|------|------|
| 消費期限前の値引き販売 | 従来は禁止していたデイリー品の値引きを解禁・拡大 | 各チェーンで本格展開中（2024年時点） |
| AI需要予測の精度向上 | 天気・イベント・曜日等を加味した発注最適化 | セブン・ファミマで大幅精度向上実績あり |
| 消費期限延長 | メーカーと連携して製造日から賞味期限を延長 | 弁当・惣菜で実績増加 |
| 恵方巻・クリスマスケーキの予約制強化 | 過剰製造の抑制 | 業界全体で標準化しつつある |
| 廃棄品の再利用・リサイクル | 廃棄食品のバイオガス化・飼料化 | 大手3チェーンで一部対応済み |

**政策的背景**: 農水省・環境省が設定した「2030年までに食品ロスを2000年比で半減」という目標が業界への圧力になっている。AIと需要予測の精度向上は規制対応と直結。

### 6-3. 新サービス領域の拡大

コンビニは「地域インフラ」としての機能を年々拡大中。

| 領域 | 主な取り組み | 主導チェーン |
|------|------------|------------|
| **金融サービス** | セブン銀行ATM（約27,000台）・ローソン銀行・ファミマTカード発行終了→ファミペイへ移行 | セブン（セブン銀行）・ローソン（ローソン銀行） |
| **ヘルスケア** | 調剤薬局併設（ローソン）・健康食品特化売場・オンライン診療サービス連携 | ローソンが最も積極的 |
| **行政サービス** | マイナンバーカード活用・住民票交付（マルチコピー機）・行政手続き代行 | 全チェーン共通（東芝テック等のマルチコピー機経由） |
| **EC・配送** | 宅配便受け取り・冷蔵ロッカー設置・当日配達拠点 | 全チェーン対応（ヤマト・ゆうパック連携） |
| **デジタルコンテンツ** | ゲームコード・音楽・映像コンテンツのコード販売 | 全チェーン共通 |
| **シェアリング** | コインランドリー・EV充電・シェアサイクルポート | ローソン・ファミマが先行 |

**地域社会との連携強化**: 2025年以降、過疎地でのコンビニを「地域の生命線」として自治体と協定を結ぶ事例が増加。高齢者への宅配（徒歩困難な顧客向け）・見守りサービス・処方薬の受け取りなど、純粋な小売の枠を超えた機能が求められている。

### 6-4. デジタル戦略の方向性（2024-2026）

| トレンド | 内容 |
|---------|------|
| **アプリ・ポイント経済圏の強化** | 各チェーンのアプリ会員数拡大競争。クーポン・スタンプ・ポイントによるリピート購買促進。2024年時点でセブンアプリが5,000万DL超、ファミペイ2,000万超 |
| **データマーケティング** | POSデータ×アプリ会員データを掛け合わせた個人向けマーケティング。メーカーへのデータ提供が新たな収益源に |
| **AI・生成AI活用** | SVの業務支援（店舗訪問前に異常値を検知してAIがレポート生成）・商品企画支援（SNSトレンド解析）・チャットボットによる加盟店サポート |
| **リアル×デジタルの融合** | 店内のデジタルサイネージ広告（ファミマが先行）・アプリクーポンと連動した購買誘導 |

## 7. PM参画にあたっての留意点

本節は、笹尾豊樹PM（2026年6月参画予定）が案件に入る際に特に意識すべき業界固有の観点を整理した。

### 7-1. コンビニ業界特有のステークホルダー構造

```
[案件関係者マップ（ストコン移行案件の場合）]

発注者（コンビニ本部のIT部門）
    ├── 商品システム部  → 商品マスタ・棚割連携に関与
    ├── 物流システム部  → 発注・DC連携に関与
    ├── 店舗オペレーション部 → 現場業務・SV活用
    └── 経営情報部      → DWH・需要予測連携

協力ベンダー
    ├── NEC・富士通等の既存ベンダー（レガシー部分を熟知）
    ├── クラウドベンダー（AWS/Azure）
    └── 加盟店（現場エンドユーザー）→ UAT参加者として重要

SIer（参画先）
    └── PMO・設計・移行支援・テスト管理
```

### 7-2. 業界固有のリスクとPMが押さえるべきポイント

| リスク | 内容 | 対策 |
|-------|------|------|
| **24時間365日稼働の制約** | 計画停止が事実上不可能。メンテナンス窓口は深夜2〜5時に限定 | 段階移行・ブルーグリーンデプロイ計画の立案が必須 |
| **加盟店への影響管理** | システム変更はオーナー（約16,000〜21,000店）の日常業務に直結 | 変更管理計画と十分なトレーニング期間の確保 |
| **廃棄ロスへの過敏性** | 発注精度の低下は加盟店収益に直接影響。現場の反発リスク | 移行前後での発注精度KPIのモニタリング設計 |
| **食品安全法・税法への対応** | 廃棄データ・精算データは法的要件を満たす必要 | 要件定義段階での法規制チェックを徹底 |
| **PCI DSS対応** | 決済システムに関わる場合、PCI DSS v4.0（2025年3月完全移行）への準拠必須 | セキュリティ要件を初期設計に組み込む（後付けは高コスト） |
| **既存ベンダーとの関係** | NECや富士通等の既存ベンダーが仕様を熟知しており、情報収集に協力が必要 | キーパーソンの早期特定と関係構築 |

### 7-3. コンビニ業界の「言語」（用語・暗黙知）

案件参画時にネイティブレベルで使えると信頼感が上がる用語・概念:

| 用語 | 概念の急所 |
|------|---------|
| **日販（にっぱん）** | 1日の売上高。店舗の健全性指標。「日販が60万円を超えるか」が重要KPI |
| **廃棄ロス** | 売れ残り廃棄の金額。加盟店収益を直撃。「廃棄ロス率を1%下げる」が現場の切実な課題 |
| **単品管理（単管）** | 個々の商品を単品レベルで需給管理する思想。コンビニの競争力の源泉 |
| **チャージ** | ロイヤリティ。加盟店が本部に支払う粗利配分 |
| **EOB/EOS** | 電子発注（EOB=Electronic Order Book、EOS=Electronic Ordering System）。ストコンから本部への発注通信 |
| **フェイス** | 商品の正面向きの陳列数。棚割の基本単位 |
| **プラノグラム** | 棚割の配置計画図。本部が作成し店舗に指示 |
| **SV（スーパーバイザー）** | 本部が店舗を管理・支援するフィールド担当者。IT施策の現場橋渡し役 |
| **デイリー品** | 賞味期限が短い商品群（弁当・おにぎり等）。在庫管理の最難関カテゴリ |
| **FF（ファストフード）** | 店内調理品（フライヤー・コーヒーマシン等）。カウンターFFとも |
| **共配（きょうはい）センター** | 複数メーカーの商品を一括配送する物流センター |
| **DC連携** | 配送センターとのシステム連携（発注→センター処理→納品確認） |
