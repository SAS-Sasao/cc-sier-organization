## 1. 調査サマリー

### 1-1. 各社のDX戦略ポジション

2024〜2026年時点のコンビニ大手3社のIT戦略は、親会社・資本構造の違いが技術選定を左右しており、単純な技術比較では捉えられない構造がある。

| 項目 | セブン-イレブン | ファミリーマート | ローソン |
|------|--------------|--------------|--------|
| 親会社 | セブン&アイ・ホールディングス | 伊藤忠商事（50.1%） | 三菱商事・KDDI（各50%） |
| IT戦略の方向性 | グループ横断の独自IT基盤を内製・自社最適化 | Microsoftとの全社DX提携でAzure軸に統合 | KDDI通信基盤+内製化（LDI）+NEC外部ベンダーの三者構造 |
| クラウドベンダー | AWS（主）+GCP（AI系） | Azure（主）| AWS+GCP（用途別） |
| IT内製化度 | 高（セブン&アイグループIT会社を持つ） | 中（Azureに依存、内製は限定的） | 中（LDIによる内製化推進中） |
| 店舗数（2025年時点） | 約21,000店 | 約16,600店 | 約14,600店 |

### 1-2. SIer視点での最重要示唆

- **セブン-イレブン**: 自社IT部門と既存ベンダー（NEC、富士通）の存在感が大きく、外部SIerの参入余地は限定的。ただしAI関連の新規PJTでは Google との協業実績あり。
- **ファミリーマート**: Microsoft との大型提携（2022年発表）で Azure が基本方針。SI パートナーとして TIS・アクセンチュア等が関与。NEC の存在感は低下傾向。
- **ローソン**: NEC が最も深く関与。ストコン移行案件は NEC の主戦場。KDDI の影響力拡大が今後の変数になる。
- **トライアル**: 小売DXのパイオニアとして独自路線（完全無人レジ・AI棚管理）を歩む。国内SIerよりも海外スタートアップとの直接協業が多い。

### 3-2. マイグレーション戦略

**セブン-イレブン**

セブン-イレブンは「ストラングラーフィグ（Strangler Fig）」パターンに近い段階的移行を2022年から推進。発注AI・需要予測機能を先行してクラウド化し、POS基幹処理は現状エッジで維持するハイブリッド構成を採用。2024〜2025年にかけてROSSの後継基盤を刷新し、AWS上のコンテナ基盤（ECS/Fargate）へのリプラットフォームを進めた。「全部まとめてクラウド」ではなく、**機能単位での段階移行**を明確な方針としている。

| 移行フェーズ | 内容 | 時期 |
|-----------|------|------|
| Phase 1 | 分析・AI機能をクラウドへ先行移行 | 2022〜2023年 |
| Phase 2 | 発注・需要予測基盤のフルクラウド化 | 2023〜2024年 |
| Phase 3 | ストコン本体のリプラットフォーム（ECS/Fargate） | 2024〜2025年 |
| Phase 4 | 全店舗（約2.1万店）への段階展開 | 2025年〜継続中 |

**ファミリーマート**

2022年のMicrosoft全社DX提携を起点に、Azureを基盤としたリアーキテクチャを推進。特徴的なのは「**シン・ストコン**（Thin Client化）」アプローチで、店舗端末をシン化してAzure上のAPI群に処理を委ねる設計を志向。2024〜2025年は発注AI・在庫管理をAzure Cognitive Services + Azure OpenAI Serviceベースで刷新した。

| 移行フェーズ | 内容 | 時期 |
|-----------|------|------|
| Phase 1 | 本部系・分析系のAzure移行 | 2022〜2023年 |
| Phase 2 | 店舗端末のシン化（Azure IoT Edge活用） | 2023〜2024年 |
| Phase 3 | 発注・在庫AIのAzure OpenAI Service活用 | 2024〜2025年 |
| Phase 4 | POS・決済系のモダナイズ | 2025年〜 |

**ローソン**

ローソンは既存調査（`wbs-2-5-1-nec-lawson-dx-strategy.md`）で詳述した通り、NEC主導でオンプレ→AWSへの移行PJTが進行中。特徴は**三者（KDDI/三菱商事/LDI）の意思決定構造**が移行戦略に影響する点。KDDIの通信基盤（閉域網）との親和性からAWSを選択し、AI系処理はGCPの精度優位性も活用するデュアルクラウド戦略。

| 移行フェーズ | 内容 | 時期 |
|-----------|------|------|
| Phase 1 | AI.CO（AI発注）のクラウド基盤整備 | 2022〜2023年 |
| Phase 2 | ストコン本体のクラウド移行設計・PJT開始 | 2023〜2024年 |
| Phase 3 | 本番移行（NEC主導・段階展開） | 2024〜2026年（進行中） |
| Phase 4 | Real×Tech Lawson基盤との統合 | 2026年〜 |

コンビニ各社はカード決済処理量の多さから「加盟店レベル1」（年間600万件超）に該当し、最高水準のPCI DSS対応が義務付けられる。

| 項目 | セブン-イレブン | ファミリーマート | ローソン |

| PCI DSS バージョン | v4.0 対応済み（2025年3月31日期限） | 同左 | 同左 |

| CDEスコープ管理 | クラウド環境でのセグメンテーション。AWS VPC設計で最小化 | Azure Virtual Network + NSG | AWS VPC + KDDI閉域網の二重境界 |
| WAF | AWS WAF | Azure Front Door WAF | AWS WAF + CDN |
| MFA | 管理者アクセスに全面適用 | Microsoft Entra ID（旧AAD）でSSO+MFA | NEC IAM + AWS IAM Identity Center |
| 年次監査 | 外部QSAによる審査 | 同左 | 同左 |

### 6-2. ベンダーエコシステムの構造的特徴

**セブン-イレブン**
グループIT会社（セブン&アイ・ネットメディア、オムニセブン）を持ち、IT内製化度が高い。外部SIerはハードウェア供給・保守が中心で、上位のアーキテクチャ設計はグループIT主導。GoogleとのAI協業（Vertex AI活用）は2023〜2024年に公開され、AWS偏重から脱却してベストオブブリード戦略に舵を切っている。

**ファミリーマート**
2022年11月のMicrosoft全社DX提携（10年・数百億円規模と推定）がエコシステムを一変させた。Azureを基盤に、Microsoft Copilot・Teams・Power PlatformでDXを推進。コンサル系SIer（アクセンチュア・デロイト）がAzure移行の実装パートナーとして関与。NEC・富士通はハードウェア保守ベンダーとしての役割が中心になりつつある。

**ローソン**
既存調査（`wbs-2-5-1-nec-lawson-dx-strategy.md`）の通り、三者構造（KDDI/三菱商事主導のReal×Tech + LDIの内製化 + NECの外部SI）が明確。NECがストコン周辺とPOS/セルフレジで主要プレイヤーの地位を維持しているが、LDIとの境界線の明確化がPM参画時の最重要課題。

## 付録: ストコン案件PMへの示唆

### PM参画時の重要確認事項（技術観点）

| 確認事項 | 理由 | 確認先 |
|---------|------|--------|
| ローソン現行ストコンのOS・ミドルウェアバージョン | 移行リスクの正確な評価のため | NEC社内・ローソンIT担当 |
| AWS選定の経緯と根拠（KDDI主導 vs NEC主導） | 「なぜAWSか」を問われた際に答えるため | プロジェクト設立文書 |
| KDDI WAKONXとのAPI連携要件 | クラウド移行後のデータ接続設計に影響 | LDI/KDDI担当 |
| LDIが内製化する機能範囲の最新状況 | NECのスコープが侵食されていないかの確認 | LDI（IT戦略窓口） |
| PCI DSS v4.0対応の現状ギャップ | 移行PJTとPCI対応が重複しないようにするため | ローソン情報セキュリティ担当 |
| ファミリーマートのAzure移行進捗（競合比較） | ローソンのベンチマーク・差別化根拠として | 公開情報・業界誌 |

### 競合チェーンの技術動向がローソン案件に与える影響

| 競合動向 | ローソン案件への影響 |
|---------|------------------|
| セブン-イレブンがAWS+GCPデュアルクラウドで先行 | 「ローソンもAWS+GCP」の組み合わせの説得力が増す。セブンの移行実績をベンチマークに活用可能 |
| ファミリーマートがAzureで実績構築 | ローソン側の「なぜAzureでなくAWSか」への回答準備が必要 |
| トライアルが内製化+スタートアップ協業で成果 | LDI（内製化推進）の発言力強化につながる。NECへの要求水準が上がる可能性 |
| コンビニ全社でPCI DSS v4.0完全対応（2025年3月完了） | ローソンのストコン移行と同タイミングで対応コストが発生しないよう工程を調整する必要 |
