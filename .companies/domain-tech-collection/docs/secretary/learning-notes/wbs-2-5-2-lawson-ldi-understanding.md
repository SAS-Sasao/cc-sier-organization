## 1. エグゼクティブサマリー

LDI（ローソンデジタルイノベーション）は2016年1月にローソン（出資比率66%）とシグマクシスの共同出資で設立され、2020年4月にローソンの完全子会社となった約123名のIT内製チームである。その設立目的は「アウトソース型からの脱却」であり、コンシューマー向けアプリ・Web・データ分析・IoT PoCの領域を内製化している。技術スタックはAWSを中心とするクラウドネイティブ構成（Go/Swift/Kotlin/Next.js）で、スクラム開発を採用する。

NEC等のSIerとの棲み分けは明確で、LDIはアプリケーション層（コンシューマー接点）、NECはハードウェア層・AI層（POS・セルフレジ・顔認証）、フューチャーアーキテクトは基幹系SIという三者分担が成立している。ストコンAWS移行はLDIの担当外（インフラ・基幹系領域）であるため、NECにとってLDIとの直接競合は起きにくいが、連携設計を誤ると関係悪化のリスクがある。KDDI経営参画後は、LDIがKDDI WAKONXとの連携を前提とした技術選定を進める可能性があり、NEC側PMとして注視が必要な変化点となっている。

## 2. LDIの設立経緯・目的・ミッション

### 2-1. 基本情報

| 項目 | 内容 |
|------|------|
| 正式名称 | 株式会社ローソンデジタルイノベーション（Lawson Digital Innovation, Inc.） |
| 設立 | 2016年1月15日（事業開始: 2016年2月1日） |
| 資本金 | 9,900万円 |
| 従業員数 | 約123名（2025年8月時点） |
| 代表取締役社長 | 三木義之 |
| 所在地 | 東京都品川区大崎一丁目11番2号 ゲートシティ大崎イーストタワー |

### 2-2. 設立の背景と目的

**核心にある問題意識**: アウトソース型のシステム開発体制から、企業主導型（内製化）のシステム開発体制への移行。

ローソンはSIer多重委託体制による「自社でシステムをコントロールできない」状態を課題として認識し、その解決策としてLDIを設立した。設立時に出資したシグマクシスはコンサルティング会社であり、PMノウハウとデジタルテクノロジーの知見を提供する役割を担った。

### 2-3. 資本構成の変遷

| 時期 | ローソン | シグマクシス | 意味合い |
|------|---------|-------------|---------|
| 2016年1月（設立時） | 66% | 34% | シグマクシスが立ち上げ支援として参画 |
| 2020年4月 | 100% | 0% | 内製化路線が軌道に乗り、シグマクシスの役割完了と解釈 |

**2020年の完全子会社化の示唆**: LDI独自の開発文化・プロセスが確立されたことで、外部コンサルへの依存なしに独立運営が可能になったと読み取れる。シグマクシスのPMノウハウはLDI内部に移転・定着済みである。

### 2-4. 沿革

- **2016年1月**: ローソン（66%）とシグマクシス（34%）の共同出資で設立
- **2016年2月**: 事業開始
- **2020年4月**: ローソンの完全子会社化（シグマクシスが全株式を譲渡）
- **2026年1月**: 設立10周年

## 4. ローソン本体との役割分担

### 4-1. ローソングループのITパートナーマップ

LDIはローソングループのIT内製チームであるが、LDI単独ですべてのITを担うわけではない。以下の多層分担構造が成立している。

| パートナー | 担当領域 | 関係性 |
|-----------|---------|--------|
| **LDI（内製）** | ローソンアプリ、Web系、データ分析、IoT PoC | ローソン完全子会社（内製チーム） |
| **NECプラットフォームズ** | POS端末ハードウェア・ファームウェア（共同開発） | ハードウェアパートナー |
| **NEC** | セルフレジ、顔認証、映像解析AI、深夜省人化実験 | AI・ハードウェアパートナー |
| **フューチャーアーキテクト** | 統合マスタシステム、Loppi、AI.CO | 基幹系SIパートナー（2007年〜） |
| **日立ソリューションズ** | AWSクラウド基盤設計・構築・運用 | クラウドインフラパートナー |
| **NSSOL（日鉄ソリューションズ）** | マネージドクラウド「absonne」（既存システム移行） | マルチクラウドパートナー |
| **アイレット（cloudpack）** | 公式Webサイト群のAWS移行、セキュリティ | WebインフラSIパートナー |
| **KDDI** | 通信インフラ、WAKONX（AI基盤）、Real×Tech | 経営パートナー（議決権50%） |
| **日本マイクロソフト** | 店舗DX（カメラ・音声データ分析AI）、Azure IoT | テクノロジーパートナー |

### 4-2. 内製 vs 外注の境界線（推定）

```
[LDI内製領域]
  ├── コンシューマー接点（アプリ、Web）
  ├── データ分析・サイエンス
  ├── IoT PoC
  └── 先進技術検証（生成AI等）

[外注（SIer委託）領域]
  ├── 基幹系システム（フューチャーアーキテクト）
  │   ├── 統合マスタシステム
  │   ├── Loppiシステム
  │   └── AI.CO（発注AI）
  ├── POS/レジ端末（NECプラットフォームズ）
  ├── クラウドインフラ（日立ソリューションズ、NSSOL）
  ├── AI・顔認証（NEC）
  └── 店舗DX実証（Microsoft、KDDI）
```

### 4-3. LDI vs NECの棲み分け

| 軸 | LDI | NEC |
|----|-----|-----|
| レイヤー | アプリケーション層（モバイルアプリ、Web） | ハードウェア層 + AI層 |
| 対象 | コンシューマー接点、データ分析 | POS端末、セルフレジ、顔認証、映像解析 |
| 開発スタイル | アジャイル、内製、スクラム | 受託開発、ハードウェア設計 |
| 技術ドメイン | Swift/Kotlin/Go/Next.js | 組込み、画像認識AI、生体認証 |
| クラウド | AWS上のアプリ開発 | オンプレ+クラウドのハイブリッド |

**重要**: ローソンにおけるNECの役割は「ハードウェア・AI特化」であり、基幹系SIの中心はフューチャーアーキテクトが担っている。これはセブン-イレブン案件（NECがフルクラウド店舗システムの構築・PMO・発注システム開発を担当）とは大きく異なる。

## 6. KDDI統合後の変化・今後の方向性

### 6-1. KDDIの経営参画が意味するもの

2024年にKDDIがローソンの議決権50%を取得し、経営に参画した。この変化はLDIの技術戦略にも影響を与える可能性がある。

- **WAKONX連携**: KDDIのDXプラットフォーム「WAKONX」との連携が前提になる技術選定が増える可能性
- **通信インフラ統合**: KDDI回線とローソン店舗システムの一体化（現行もKDDI IP-VPNを使用）
- **顧客データ基盤の高度化**: KDDI会員データとローソン会員データの統合により、LDIのデータサイエンス部門の分析基盤に変化

### 6-2. 今後の注力領域（採用要件・テックブログから推定）

| 領域 | 推定根拠 |
|------|---------|
| クラウドネイティブ化の加速 | AWS Lambda、DynamoDB等サーバーレス技術の活用拡大 |
| 生成AI活用の本格化 | Cline for LDI、LiteLLM基盤の展開 |
| セキュリティ強化 | セキュリティエンジニア専任ポジションの新設 |
| ITアーキテクチャの刷新 | マルチクラウド（AWS/Azure/GCP）対応アーキテクト採用 |
| 品質管理の高度化 | 品質管理マネージャーの採用 |
| データ駆動経営 | データサイエンティストの増員 |

### 6-3. 未解明の論点（ソースレポート記載）

以下はLDI実態分析の段階で未確認となっている論点であり、参画後に情報収集が必要な項目である。

1. LDIの売上規模・利益構造
2. シグマクシス撤退の真の理由（戦略的判断か、関係悪化か）
3. LDIのローソン本社ITソリューション本部との権限関係
4. ストコンの現行ベンダー（2019年刷新時のハードウェアベンダー）
5. NEC案件の具体的スコープ（ストコンAWS移行においてNECがどこまでを受注しているか）
6. KDDI経営参画後のIT戦略変化（KDDIのAI基盤がLDIの技術選定に与える影響）

## 7. ストコン移行案件との関連

### 7-1. LDI理解がPMにとってなぜ重要か

ストコンAWS移行においてLDIは直接の担当ベンダーではないが、**技術要件・スコープ定義・API連携設計の実質的な決定者**になる可能性が高い。LDIの実態を正確に把握することが、案件を円滑に進めるための前提条件となる。

### 7-2. PM視点での5つの活用ポイント

**ポイント1: NECの立ち位置を正確に把握する**

ローソンにおけるNECの役割はセブン-イレブン案件（フルクラウド店舗システムの中核SI）とは大きく異なる。ローソンではNECは「POS端末・セルフレジ等のハードウェア + AI技術」のパートナーであり、基幹系SIの中心はフューチャーアーキテクトが担っている。ストコンAWS移行案件がNECに発注された場合、それはNECにとってローソン案件の中でSI領域への拡大を意味する可能性がある。

**ポイント2: LDIとの協業ラインを見極める**

LDIは約123名の内製チームで、主にコンシューマー向け（アプリ・Web）を担当する。クラウドインフラや基幹系は外部パートナーに委託している。ストコンAWS移行は**LDIの担当外**（インフラ・基幹系領域）である可能性が高く、直接競合よりも**連携・協調**が求められる関係性になる。LDIのAPIとのデータ連携設計や、LDIが使用するAWS基盤との整合性を早期に確認することが重要。

**ポイント3: フューチャーアーキテクトとの関係に注意する**

フューチャーアーキテクトは2007年からローソンの基幹系SIを担当する長年のパートナー。統合マスタ、Loppi、AI.CO等の主要システムを握っている。ストコン移行においても、マスタデータ連携やAI.COとのデータ統合が必要になるため、フューチャーアーキテクトとの協調は必須となる。

**ポイント4: マルチクラウド環境を理解した上で移行設計を進める**

ローソンはAWS（新規開発）+ NSSOL absonne（既存移行）+ Azure（IoT）+ KDDI WAKONX（AI基盤）のマルチクラウド戦略を採用している。ストコン移行では既存システムとの接続・データ連携において、この複雑なマルチクラウド環境の全体像を把握した設計が必要。特にKDDI WAKONXとの連携要件は、KDDI経営参画後の新たな制約条件となる可能性がある。

**ポイント5: シグマクシス出身文化への対応**

LDI設立時のシグマクシスのPMノウハウはLDI内部に移転・定着している。コンサル出身のメンバーが多い組織であり、「なぜこのアーキテクチャか」「なぜNECか」を論理的に問い詰める文化を持つ可能性がある。PM（笹尾）として、意思決定の根拠を常にドキュメント化しておくことが信頼獲得につながる（ADR形式の活用を推奨）。

### 7-3. LDI担当システム vs NEC担当システムの整理

| システム | 担当 | ストコン移行との関連 |
|---------|------|------------------|
| ローソンアプリ（iOS/Android） | LDI内製 | 移行後の店舗データとのAPI連携先 |
| データ分析基盤（SAS/Python/R） | LDI内製 | 移行後の売上・在庫データの分析先 |
| POS端末（ハード・ファーム） | NECプラットフォームズ | ストコンとのデータ連携が必要 |
| セルフレジ | NEC | ストコンとのデータ連携が必要 |
| 統合マスタシステム | フューチャーアーキテクト | 移行後のマスタデータ連携先 |
| AI.CO（発注AI） | フューチャーアーキテクト | 移行後の発注データ連携先 |
| AWSクラウド基盤 | 日立ソリューションズ | 移行先クラウド基盤の協業パートナー |
| ストアコンピュータ | 不明（NEC/富士通推定） | 移行対象本体 |

## 10. 理解度セルフチェック

以下の5問に答えて、理解度を確認してください。回答例はその下に記載します。

---

**Q1. LDIの設立目的と、2020年にシグマクシスが株式を手放した意味を説明してください。**

<details>
<summary>回答例</summary>

設立目的は「アウトソース型のシステム開発体制から、企業主導型（内製化）のシステム開発体制への移行」。シグマクシスはLDI立ち上げを支援するコンサルとして34%出資したが、内製化路線が軌道に乗りLDI独自の開発文化・プロセスが確立されたことで、2020年4月に全株式をローソンへ譲渡した。これはシグマクシスの「立ち上げ支援」の役割が完了したことを意味する。
</details>

---

**Q2. ローソンのIT協力会社エコシステムにおける、LDI・NEC・フューチャーアーキテクトの役割分担を説明してください。**

<details>
<summary>回答例</summary>

- **LDI**: コンシューマー接点（アプリ・Web）・データ分析・IoT PoCの内製
- **NEC**: POS端末・セルフレジ等のハードウェア層 + 顔認証・映像解析AIのAI層
- **フューチャーアーキテクト**: 2007年から統合マスタシステム・Loppi・AI.CO等の基幹系SIを担当

NECはローソンにおいて「ハードウェア・AI特化」の役割であり、セブン-イレブン案件（基幹SIの中核）とは性質が異なる。
</details>

---

**Q3. ストコンAWS移行案件において、LDIが「競合」ではなく「連携先」になる理由を説明してください。**

<details>
<summary>回答例</summary>

LDIの内製領域はコンシューマー向けアプリ・Web・データ分析であり、クラウドインフラや基幹系は外部パートナーに委託している。ストコンAWS移行は「インフラ・基幹系領域」に該当するため、LDIの担当外である可能性が高い。したがってNECとLDIは直接競合しにくく、むしろ移行後のAPI連携・データ連携において協調関係が求められる。
</details>

---

**Q4. ローソンのマルチクラウド構成（4種）を列挙し、ストコン移行との関係を説明してください。**

<details>
<summary>回答例</summary>

- **AWS**: 新規開発システム全般（日立ソリューションズ、アイレットが担当）
- **NSSOL absonne**: 既存業務システムのリフト移行先
- **Azure**: IoT在庫管理実証、店舗DX（カメラAI）
- **KDDI WAKONX**: AIサービス基盤、会員データ統合

ストコン移行では移行先のAWSに加え、既存のabsonne・Azure・WAKONXとの連携設計が必要となる。単なるサーバー移転ではなく、既存マルチクラウド環境との整合性を取った設計が求められる。
</details>

---

**Q5. KDDI経営参画（議決権50%取得）がNEC側PMの視点でリスクになりうる理由を説明してください。**

<details>
<summary>回答例</summary>

KDDIが独自のDXプラットフォーム（WAKONX）を持ち、通信インフラからデータ分析まで提供できる立場にあるため、ローソンの技術選定においてKDDI主導の方向性が強まる可能性がある。特に、KDDIが独自クラウド基盤を推進した場合、AWS選定が見直されるリスクがある。また、LDIがKDDI WAKONXとの連携を前提とした技術要件を定義する場合、NECの設計アプローチにも影響が生じる。PM（笹尾）としてはステークホルダー分析でKDDI側の意向を早期確認することが重要。
</details>
