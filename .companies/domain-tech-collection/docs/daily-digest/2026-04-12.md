## C. クロスドメイン分析

### 1. Claude Skills/MCP の実装爆発とエンタープライズ適用の加速

本日の技術側は Claude Code Skills/MCP 関連記事が突出している。デザインシステムの Skill 化、社内業務 MCP サーバー群の一挙公開、Obsidian/3D CAD との連携、superpowers プラグインで開発フローを強制する設計パターン、permissions の優先順位検証、メモリ仕様の実測まで、個別の Tips から組織適用の知見までが同日中に並んだ。AWS Agent Registry のプレビュー公開と AgentCore Browser の OS 操作対応が揃ったことで、「個別開発者の工夫」フェーズから「企業での集中ガバナンス」フェーズに移行する兆しが見える。SIer としては、既存のシステム接続インターフェースを MCP サーバーとして再パッケージ化する提案、Skill を通じた社内設計標準の AI 強制実装など、顧客の AI 導入ロードマップの次の一手を具体化しやすいタイミング。特にローソンのストコン AWS 移行 PoC には、運用手順書を Skill 化して運用チームと共有する仕組みが親和性高い。

### 2. エージェンティックコマースの実装フェーズ化と小売基幹への波及

Amazon「Help Me Decide」、LINE ヤフー Yahoo!ショッピングの AI エージェント、Visa/Mastercard のエージェント型決済が同時並行で実装段階に入り、EC 顧客体験の主役が「検索＆絞り込み UI」から「AI への自然言語相談」に移る兆候が濃厚になってきた。これは POS/EC 側だけの話ではなく、商品マスタ・価格・在庫・決済・物流 API の一貫性が前提になるため、基幹系にまで波及する。SIer としては、エージェントから呼び出される形の「商品情報 API」「在庫引当 API」「セーフティ決済 API」の設計と、エージェントが呼び出す際のレート制限・認証・監査ログの整備が新規スコープとして立ち上がる。流通業界の DX 相談窓口を持つベンダーなら、既存顧客のリアーキテクチャ提案にこの観点を組み込むだけで差別化要因になる。

### 3. ドラッグストア／SM 業界の再編とシステム統合の大型案件化

ツルハHDとウエルシアHDの統合で 2 兆円グループ誕生、ツルハHD単体でも 3 年 450 店舗の新規出店、イオンの 26 年度 GMS/SM 構造改革、食品スーパー業界の M&A 頻発、伊藤忠食品 TOB 成立と、小売業界の再編は決算好調を背景に一気に進行している。この規模の統合では、POS/WMS/MDM/EC/CRM の基幹系を合併後 2〜3 年で統合する必要があり、ベンダー/SIer にとっては数百億円規模の統合プロジェクトが立ち上がる。特にツルハ×ウエルシアのようなケースでは、両社が独自に構築した在庫管理・マスタ設計のギャップを埋めるデータ統合が最重要課題になるため、dbt platform の Developer agent や Microsoft Fabric の容量設計ノウハウといった技術側の進展が直接効く。リアルタイムな在庫共通化と顧客 ID 統合が PoC の早期テーマになる。

### 4. 流通 ISAC 設立とサプライチェーンセキュリティの一体運用

製造・卸・小売横断の「流通ISAC」設立は、トライアル・アサヒ・NTT が牽引する業界横断のセキュリティ共助で、技術側の supply chain 攻撃の話題（GitHub/npm 防御設定 8 選、minimumReleaseAge と audit override の両立、VSCode 拡張経由の攻撃事例）と呼応する。小売業は EDI・API・POS・物流 WMS・EC が相互接続されるため、どこか一点の脆弱性がサプライチェーン全体に波及するリスクを以前から抱えてきた。SIer として流通業界に入っていく際には、提案時点で「流通ISACのレポート」を顧客と共有できる体制を整え、「SOC + EDR + SBOM 運用 + 依存パッケージ minimumReleaseAge」という 4 点セットをベースラインとして提示することが、DX 提案と一体で進める現実解になる。

### 5. データ基盤の AI 化と「セマンティックレイヤー経由の分析エージェント」設計思想
