## C章 クロスドメイン分析

### 1. AIツールのセキュリティと EC 信頼基盤

CursorにCVSS 9.9の脆弱性が発見された事案（A6-1）は、AI駆動開発ツールがサプライチェーンの新たなリスクベクトルになり得ることを示している。同時に小売ドメインでは、LINEヤフーの転売対策ポリシー策定（B6-3）やカード不正利用額の11年ぶり減少（B6-2）など、ECプラットフォームの信頼性強化が進展している。SIerとしては、AI開発ツールのセキュリティ監査フレームワークの整備と、EC事業者向け不正検知・転売防止ソリューションの提案を両輪で進めることが求められる。AIエージェントに認証情報を安全に渡す手法（A1-4）の知見は、EC決済基盤のセキュリティ設計にも応用可能である。

### 2. 生成AI投資の加速と小売業のAI活用

Anthropicの評価額141兆円報道（A2-8）やSageMaker AIのエージェント体験発表（A2-2）は、生成AIへの企業投資が急速に拡大していることを物語る。小売ドメインではZOZOのAIコーディネート提案「ラボくん」（B4-2）やメイキップのサイズ再現AI画像生成（B4-8）など、具体的なAI活用事例が出始めている。SIerは、Bedrock AgentCoreのGovCloud対応（A2-3）やSAP MCP Server GA（A2-4）といったエンタープライズ向けAI基盤の導入支援に加え、小売業特化のAI活用PoC（パーソナライゼーション、商品画像生成、在庫最適化）を提案していくことが差別化につながる。

### 3. 物流危機とサプライチェーン・レジリエンス

ホルムズ海峡危機に伴う航空貨物運賃65%高止まり（B5-10）と米中制裁衝突の影響（B2-8）は、グローバルサプライチェーンの脆弱性を改めて浮き彫りにしている。一方で、花王・三菱食品の共同配送コンソーシアム「CODE」（B2-7）や三協の短時間勤務者対応の誤出荷ゼロ物流モデル（B4-13）など、国内では物流効率化の具体的な取り組みが進んでいる。SIerには、EventBridgeのデータプレーンログ対応（A3-10）やCloudWatch Logs Insightsの新機能（A3-14）を活用した物流オペレーションの可観測性強化基盤や、デジタル技術活用の政府補助金（B4-11）を組み合わせた物流DX提案が求められる。

### 4. データプラットフォーム進化と小売分析基盤

dbt Fusion GA（A4-1）、AWS Glue 5.1でのApache Iceberg対応（A4-2）、QuickSightのS3 Table Bucket対応（A4-3）と、データ基盤の選択肢が急速に広がっている。小売領域ではアーバンリサーチの基幹システム刷新によるOMO在庫一元管理（B4-3）や、GMO調査の自社EC購買単価5年間136.6%成長（B5-9）など、統合データ分析のニーズが高まっている。SIerとしては、Iceberg + dbt + QuickSightの組み合わせで小売業のメダリオンアーキテクチャを構築し、リアルタイムOMO分析・顧客単価予測を実現する提案が有効である。CloudWatch LogsのS3テーブル統合（A3-3）も運用ログ分析の高度化に活用できる。
