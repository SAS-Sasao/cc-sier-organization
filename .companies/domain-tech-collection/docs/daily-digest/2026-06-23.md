## C. クロスドメイン分析

### 1. AIエージェント開発基盤の成熟と小売EC事業者への示唆

技術側ではAWS Lambda MicroVMs、Amazon MSK Agent Skills、GitHub CLI Agent Skills一覧表示など、AIエージェント実行基盤が着実に整備されている。同時にGoogle Workspace StudioのGA化やClaude Cowork、Codexの農業DX事例が示すように、非エンジニアがAIエージェントを業務に組み込む事例が急増している。小売側ではECのミカタが「AIが推薦できる情報設計」の重要性を指摘しており、今後EC事業者はAIエージェントが商品を「読める」構造化データ整備が競争力の源泉となる。SIerとしては小売クライアントへの構造化データ基盤導入提案が差別化要因になりうる。

### 2. サカナAI「Fugu」とモデルオーケストレーションの実用化

サカナAIの「Fugu」が複数フロンティアモデルを束ねるサービスとして正式リリースされ、DevelopersIOで即座に検証記事が公開された。PLaMo 3.0 Primeの登場と合わせ、国産LLMの選択肢が充実してきている。小売業界で懸念される顧客データの国外流出リスクに対し、国産LLMの活用は有力な解決策となる。SIerとして顧客のデータガバナンス要件に応じた最適なLLM選定と、マルチモデルオーケストレーション設計の提案機会が広がる。

### 3. データカタログのセマンティック検索と小売DXの接点

Glue Data Catalogにビジネス用語辞書とセマンティック検索が追加されたことで、技術者でない業務部門でもデータ発見が可能になりつつある。小売業では部門ごとにデータの呼称が異なる（「売上」と「精算額」と「レジ通過額」など）問題が根深く、セマンティック検索による用語の揺れ吸収は実務的価値が高い。DWH基盤の導入提案時にデータカタログのセマンティック層を組み込むことで、導入後の「使われないDWH」問題の回避策を示せる。

### 4. セキュリティ課題の横断的深化

技術側ではHonoのJWT脆弱性、AIスクレイピング対策「Anubis」、iOSフィンガープリンティングなど多層的なセキュリティ課題が浮上。小売側では本日のB6セキュリティ記事こそなかったものの、ヨドバシカメラのOMO展開やNETSEAのハッシュタグ自動生成など顧客接点のデジタル化が加速しており、攻撃面の拡大が懸念される。SIerとして小売クライアントのOMO/EC基盤構築時にセキュリティ・バイ・デザインの提案を組み込む必要性が高まっている。
