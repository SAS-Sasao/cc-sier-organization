## C. クロスドメイン分析

### 1. AIエージェント開発の「組織化」が小売DX推進チームに示すもの

技術側では「個人のプロンプト術をやめてチームで回るAI開発ループを作った」「AIコードレビューは人ではなく仕組みに向けろ」といった、AI駆動開発を属人的なスキルから組織プロセスへ昇格させる議論が活発化している。小売側ではYahoo!ショッピングの「AIお買い物メモ」やCaccoのAI不正検知強化など、AIを業務フローに組み込む動きが加速している。SIerとしては、小売企業のDX推進部門に対して「個人のAI活用」から「組織としてのAI運用プロセス設計」への移行を提案することが差別化ポイントとなる。Claude Code SkillsやMCP連携のナレッジを活用した、小売業向けAIオペレーション標準化パッケージの開発が有望である。

### 2. CFC併設型店舗とフルフィルメント基盤のクラウド設計

イオン八王子滝山のGreen Beans CFC併設型新店は、実店舗とネットスーパーのフルフィルメントを一体化する新モデルである。技術側ではCloudWatch Metrics CentralizationやLog Analyticsの強化が発表され、マルチサイト運用の可観測性が向上している。SIerとしてはCFC併設店舗のリアルタイム在庫同期・ピッキング最適化システムをAWS上に構築し、CloudWatchの新機能でオペレーション品質を可視化する提案が考えられる。特にECS Managed InstancesとFargateの使い分けは、店舗内エッジ処理とクラウド集約処理のハイブリッド設計に直結する知見である。

### 3. 商業動態統計にみる業態別DX投資余力とデータ基盤需要

5月の商業動態統計では家電量販27.5%増、ドラッグストア7.3%増、百貨店7.6%増と全業態が前年を上回った。好業績企業ほどDX投資余力があり、データ基盤構築の需要が見込める。技術側ではDatabricksのLTAPアーキテクチャがOLAP×OLTPを統合し、SnowflakeのData Classificationで機密データの自動分類が可能になるなど、データ基盤の成熟が進んでいる。SIerとしては好調な家電量販やドラッグストアチェーンに対して、POSデータ×顧客データの統合分析基盤をLTAP/Snowflakeアーキテクチャで提案する絶好のタイミングである。

### 4. セキュリティ対策の「コード〜決済」一気通貫と小売サプライチェーン防御

技術側では「.envがDockerイメージに焼き込まれる」脆弱性やAWS Security Hub×Microsoft Sentinelのクロスクラウド監視事例が話題となった。小売側ではCaccoの決済前後の属性・行動分析によるAI不正検知強化が報じられている。SIerとしては開発段階のシークレット管理からECサイトの決済不正検知まで、一気通貫のセキュリティ設計を小売企業に提案できる。特にuvのマルウェアチェック機能やCognitoのCMK対応など、サプライチェーン全体のセキュリティ強化ツールの組み合わせが重要になる。
