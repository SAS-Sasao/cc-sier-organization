# WBS: 提案書・見積作成プロセス標準化

## プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | 提案書・見積作成プロセス標準化 |
| オーナー | 笹尾豊樹 |
| 目的 | 引合→提案書→見積→PJ計画の属人化を解消し、品質・透明性を標準化 |
| 対象 | 大規模（5000万〜数億）× 新規開発 × 新規顧客 |
| 期間 | 約15週間（2026-03-24 〜 2026-07-04 想定） |
| 承認者 | スキル標準化チームPM |

## リスク

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|---------|------|
| SharePointのアクセス権限不足 | 高 | 中 | 事前に情シスに権限申請 |
| PM承認が遅延（ゲート1/2） | 高 | 中 | 事前相談で方向性を合わせておく |
| パイロット対象の引合がない | 中 | 低 | 過去案件で擬似的に試行 |
| 案件PMの協力が得られない | 中 | 中 | 上長経由で協力依頼。メリットを明示 |
| テンプレートが現場に合わない | 中 | 中 | パイロットで早期検証。フィードバック重視 |
