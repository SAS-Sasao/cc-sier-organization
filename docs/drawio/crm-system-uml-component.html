概要

CRM基幹システムをクラウド上で運用するための、ベンダー非依存のUMLコンポーネント図。AWS / Azure / GCP いずれのクラウドでも実装可能な抽象レイヤーで設計している。5層のレイヤー構成（Client / Edge Security / Application / Data / Operations）でMulti-AZ冗長化・VPN閉域接続・読み書き分離DBを実現する。

データ層ではDB（シリンダー図形）・ファイルサーバー（シリンダー図形）・キャッシュ（六角形）を UML標準の図形で表現し、各レイヤーの役割を視覚的に明確化している。

設計のポイント

  • クラウドベンダー非依存: AWS/Azure/GCPの固有サービス名を使わず、汎用的な名称（RDBMS、Load Balancer、Object Storage等）で記述。クラウド選定前の提案段階で使える。
  • UML図形による役割の明確化: DB・ストレージはシリンダー、アプリケーションはコンポーネント図形、キャッシュは六角形で表現し、図形を見るだけで要素の性質がわかる。
  • 5層レイヤーで責務分離: Client→Edge→App→Data→Opsの5層構成で、各層の責務を明確に分離。セキュリティ境界・スケーリング単位・運用責任が一目で把握できる。
  • Multi-AZ・読み書き分離: App ServerをAZ-a/AZ-cに冗長配置し、DBはWriter/Readerを分離。単一障害点の排除と読み取り性能の向上を両立。
