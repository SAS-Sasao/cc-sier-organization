概要

本図は、2026年6月のPM参画から2029年3月までの34ヶ月間にわたる コンビニエンスストア チェーン（約14,600店舗規模 / ローソン想定）の 店舗コンピューター（ストコン）オンプレ → AWS 移行プロジェクトの統合スケジュールをガントチャート形式で可視化したものです。

既存の学習ノート3本（WBS 4-1-1 大規模移行PMの全体像、 WBS 5-2 AWS移行ベストプラクティス、 WBS 4-2-1 コンビニ/小売業界PM特有事項）でそれぞれ個別に語られていた期間感（MAP 3フェーズ / ロールアウト3フェーズ / wave 7段階）を 1本の時間軸に統合して描いています。コンビニ業界特有の凍結期間（年末年始・GW・お盆・決算）も明示しています。

対象スコープ: Assess → Mobilize → Pilot(wave 0) → Wave 1-7 → Hypercare/Operate の全 11 フェーズ。

構成要素

要素名種類説明
Assess（評価・計画）フェーズ2026/06-2026/11（6ヶ月）。現行ポートフォリオ棚卸、6R判定、TCO試算、Landing Zone 設計。PM 参画と同期。
Mobilize（基盤構築）フェーズ2026/12-2027/03（4ヶ月）。CCoE 立ち上げ、Landing Zone 構築、移行工場整備、運用設計。
Pilot（wave 0）フェーズ2027/02-2027/04（3ヶ月、Mobilize 末尾と重なる）。10-50店規模の試験移行。運用手順・切り戻し手順検証。
Wave 1-7ロールアウト2027/05-2029/01。各 wave 約2,000店 × 7 = 14,000店。3ヶ月サイクル（1ヶ月移行＋1ヶ月安定化＋1ヶ月バッファ）。
Hypercare / Operateフェーズ2029/02 以降。全店 Go-Live 後の集中監視期間 → 定常運用（FinOps、Well-Architected review）への移行。
年末年始凍結（赤バンド）凍結期間12月下旬～1月上旬、コンビニ業界では変更凍結。2027/12-2028/01、2028/12-2029/01 の 2 回が wave 設計に影響。
軽凍結（黄バンド）凍結期間GW（5月）、お盆（8月中旬）、決算（2月、小売）は展開禁止。wave 配置時に注意すべき期間。
PM参画マーカーマイルストーン2026/06 に笹尾氏がNEC経由で案件参画。Assess フェーズの開始と同期。
全店 Go-Live マーカーマイルストーン2029/01 に全 wave 完了、全店 AWS 稼働開始。Hypercare 期間に移行。

設計のポイント

  • 総期間34ヶ月のうち実稼働は約27ヶ月: 年4回（年末年始・GW・お盆・決算）の凍結期間を除くと稼働月は年8ヶ月に縮む。 このため 5-2 ノートで見積もった「30ヶ月」は凍結を考慮するとタイトで、34ヶ月が現実的な線。 wave 3, 6, 7 は凍結期間と接触するため色を濃くして視覚的に警戒。
  • Pilot は Mobilize の末尾と重ねて配置: Landing Zone が完成してから Pilot を打つのが鉄則。 5-2 ノートは「M1-6 パイロット」と記述していたが、実際は Mobilize 後（M8-10）が適切。 このズレを図上で解消し、Pilot は wave 0 として独立表示した。
  • PM参画（2026/06）と Assess 開始を同期: 4-1-1 ノートの「参画直後のキャッチアップ」を個人活動ではなく Assess フェーズのPM稼働として再定義。 これにより「オーナーは参画月からプロジェクトに対して何をすべきか」が1本の時間軸で追える。
  • wave 店舗数は均等配分（約2,000店 × 7）: 実務では地域・業態・売上規模でばらつくが、本図は「時間軸で見た負荷平準化」の見取り図として均等配置。 実際の wave 設計時は 4-2-1 ノート 6章の 「wave7段階設計案」に従って分布を調整する。
使い方のヒント: このガントは「合意形成の初期素材」であり、確定スケジュールではありません。 参画後の Assess フェーズで 6R判定の比率・1 wave あたり店舗数・ 凍結期間の厳格度 を実測し、本図を基に wave 設計を具体化するのが想定フロー。
