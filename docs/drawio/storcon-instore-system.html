概要

コンビニエンスストアの店舗内システム構成を表現したUMLコンポーネント図。ストアコンピューター（ストコン）を中心ハブとし、POSレジ端末、マルチ決済端末、EOS/EOB（電子発注・電子受発注）による本部通信、温度管理モニタリングなど、店舗運営に必要な全要素を網羅。

店長・スタッフ・配送ドライバー・顧客の4種のアクターが関与し、ハンディターミナルによる検品・棚卸、デジタルサイネージによる価格表示も含む。ストコンは本部クラウドからマスタデータ（PLU・価格）を受信し、売上ジャーナルと温度ログを本部に送信する双方向通信を担う。

設計のポイント

  • ストアコンピューターをハブとする設計: ストコンが全モジュール（EOS/EOB、発注、在庫、検品、精算、温度管理）を統合制御する。POS端末や各種端末は直接本部と通信せず、必ずストコンを経由する。これにより店舗内の通信を一元管理し、本部回線の障害時もローカルで業務継続が可能。
  • EOS/EOBの双方向通信: EOS（Electronic Ordering System）で店舗から本部へ発注データを送信し、EOB（Electronic Order Book）で本部から店舗へ推奨発注量・新商品情報を受信する。この双方向通信により、店舗の自律的発注と本部のマーチャンダイジング指示を両立させる。
  • マルチ決済の統合: 現金・クレジットカード・ICカード（Suica等）・QRコード（PayPay等）・電子マネー（nanaco等）の5種類以上の決済手段を決済端末で統合。POS端末は売上登録に集中し、決済処理は専用端末に分離することで、決済規格の追加・変更時の影響範囲を最小化する。
  • 温度管理による食品安全の担保: 冷蔵・冷凍庫の温度を1時間ごとに自動記録し、異常温度を検知した場合は店長へアラート通知。食品衛生法に基づく温度記録の義務化に対応し、紙の温度記録簿をデジタル化する。

学習ポイント

  • コンビニ3層構造（本部-ストコン-POS）の役割分担 — コンビニチェーンのITシステムは「本部クラウド（中央サーバー）」「ストアコンピューター（店舗サーバー）」「POS端末（レジ）」の3層で構成される。本部はマスタデータ管理・全店売上分析・マーチャンダイジングを担い、ストコンは店舗内のデータ集約・発注・在庫管理のハブとして機能する。POSは販売登録に特化し、商品マスタはストコンからキャッシュを受け取る。この3層分離により、各層が独立してスケールでき、本部回線障害時も店舗が自律的に営業継続できる。
  • EOS/EOBの仕組み（Electronic Ordering System / Electronic Order Book） — EOSは店舗から本部への電子発注で、店長がストコンで発注数を入力すると本部の受注システムへリアルタイム送信される。EOBは本部から店舗への推奨発注帳で、過去の販売実績・天候・イベント情報をもとにAIが最適発注量を提案する。日本のコンビニでは1970年代にセブン-イレブンが先駆けて導入し、現在は全チェーンの標準インフラとなっている。
  • POS-ストコン間のマスタ同期とPLUキャッシュの必要性 — PLU（Price Look-Up）コードは商品のバーコードに紐づく価格・名称情報で、POS端末が高速にスキャン処理するためにはローカルキャッシュが不可欠。本部で商品マスタが更新されるとストコン経由で各POS端末にプッシュ配信される。1店舗あたり約3,000SKUのマスタを保持し、新商品投入は週次で100品以上になるため、差分同期の仕組みが重要。同期失敗時は前回マスタで営業を継続する縮退運転設計が必須。
  • マルチ決済の複雑性（現金・クレジット・IC・QR・電子マネー） — コンビニの決済手段は現金、クレジットカード（VISA/Master/JCB等）、交通系IC（Suica/PASMO等）、QRコード（PayPay/d払い/楽天ペイ等）、流通系電子マネー（nanaco/WAON等）と多岐にわたる。各決済ブランドごとに通信プロトコル・手数料率・精算サイクルが異なるため、POS端末と決済端末を分離し、決済端末側で各ブランドのプロトコル変換を吸収する設計が主流。新規決済手段の追加時にPOSソフトウェアの改修が不要になる。
  • 温度管理の法的要件（食品衛生法との関連） — 2021年6月施行の改正食品衛生法でHACCP（Hazard Analysis and Critical Control Points）に沿った衛生管理が全食品事業者に義務化された。コンビニでは冷蔵（0-10度C）・冷凍（-18度C以下）の温度を定期的に記録し、逸脱時の是正措置を文書化する必要がある。ストコンの温度監視モジュールはこの法的要件をデジタルで充足し、保健所の監査時にログを即時提出可能にする。紙の記録簿からの移行で記録漏れ・改ざんリスクも排除される。
